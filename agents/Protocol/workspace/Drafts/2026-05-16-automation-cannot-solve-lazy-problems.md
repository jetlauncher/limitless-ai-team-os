# Main Issue

## Opening insight

ประโยคหนึ่งจาก transcript ที่ผมอยากให้เจ้าของธุรกิจจำไว้คือ:

**“Automation Cannot Solve Your Lazy Problems.”**

แปลแบบตรงไปตรงมา:

ถ้า workflow ของบริษัทยังไม่ชัด AI จะไม่ทำให้มันดีขึ้น

มันจะทำให้ความมั่ววิ่งเร็วขึ้น

นี่คือจุดที่หลายบริษัทเริ่ม AI ผิดทาง

เขาเริ่มจากถามว่า:

“ตอนนี้ tool อะไรดีที่สุด?”

คำถามนี้ไม่ผิด

แต่มันยังเร็วเกินไป

คำถามที่ดีกว่าคือ:

**งานนี้ควรถูกออกแบบใหม่อย่างไร ถ้า AI ทำงาน 80% และคนอนุมัติ 20%?**

## The shift

ช่วงแรกของ AI คนส่วนใหญ่ใช้มันเป็นผู้ช่วย

เขียน caption
สรุปประชุม
แปลภาษา
ทำสไลด์
ค้นข้อมูล

มีประโยชน์ครับ

แต่ยังไม่ใช่ operating system

การเปลี่ยนจริงเกิดขึ้นตอนที่ AI ไม่ได้แค่ “ช่วยคนทำงาน”

แต่เข้าไปเป็นส่วนหนึ่งของ workflow

เช่น:

- lead ใหม่เข้ามาใน LINE OA
- AI อ่าน context และจัดกลุ่มความต้องการ
- AI ร่างคำตอบหรือ next step
- คนตรวจเฉพาะเคสที่เสี่ยง
- ข้อมูลถูกส่งเข้า CRM / Airtable / Notion
- เจ้าของเห็น dashboard ว่า lead ตกตรงไหน

นี่ไม่ใช่การใช้ ChatGPT เก่งขึ้น

นี่คือการออกแบบงานใหม่

จาก transcript เรื่อง Claude Skills มีตัวอย่างที่ชัดมาก:

Skill ไม่ใช่ prompt สวย ๆ

มันคือ SOP ที่บอกว่า:

- input คืออะไร
- ต้องใช้ tool ไหน
- step 1, 2, 3 ทำอะไร
- output ต้องหน้าตาแบบไหน
- ใครเป็นคน approve

พูดง่าย ๆ:

**Skill คือ SOP ที่ AI อ่านออกและทำตามได้**

นี่คือ shift สำคัญสำหรับเจ้าของธุรกิจไทย

AI ไม่ได้แพ้ชนะกันที่ใครรู้จัก tool เยอะกว่า

แต่แพ้ชนะกันที่ใครเปลี่ยนงานซ้ำให้กลายเป็น workflow ที่ AI ทำซ้ำได้ก่อน

## What it means for Thai business owners

บริบทของธุรกิจไทยมีปัญหาเฉพาะตัว

งานจำนวนมากไม่ได้อยู่ในระบบ

มันอยู่ใน:

- LINE chat
- หัวหน้าทีมคนเดียว
- Excel ที่ไม่มี owner ชัดเจน
- SOP ที่เคยเขียนไว้แต่ไม่มีใครเปิด
- ความจำของเจ้าของ
- วิธีทำงานแบบ “ถามพี่คนนั้น”

ถ้าบริษัทเป็นแบบนี้ การเอา AI เข้าไปทันทีจะไม่ใช่ automation

มันจะกลายเป็นการโยนความไม่ชัดให้ AI เดา

และ AI เดาเก่งมาก

ปัญหาคือมันเดาผิดได้เร็วมากเหมือนกัน

เพราะฉะนั้น ก่อนถามว่า “ควรใช้ ChatGPT, Claude, Gemini, Manus, Make, Airtable, n8n หรืออะไรดี?”

ให้ถามก่อนว่า:

**งานไหนในบริษัทที่เกิดซ้ำทุกสัปดาห์ ใกล้เงินที่สุด และยังพึ่ง hero อยู่?**

ใกล้เงิน หมายถึง:

- ตอบ lead
- follow-up ลูกค้า
- ทำ proposal
- สรุป requirement
- ทำใบเสนอราคา
- สรุป meeting action items
- เช็ค invoice
- เตรียม content ที่พาไปขาย
- สรุป feedback ลูกค้า

เริ่มตรงนี้ก่อน

ไม่ใช่งานที่ดูว้าวที่สุด

แต่งานที่ถ้าทำดีขึ้น จะทำให้เงินเข้าเร็วขึ้น ลูกค้ารอน้อยลง หรือทีมไม่ต้องถามเจ้าของทุกเรื่อง

## The operator move

สัปดาห์นี้ทำ **60-Minute AI Workflow Audit**

ไม่ต้องซื้อ tool ใหม่

เปิด blank doc แล้วเขียน 4 ช่องนี้

**1. Recurring task**

เลือกงานซ้ำ 10 งานในบริษัท

ให้คะแนน 1-5 ใน 3 เรื่อง:

- เกิดบ่อยแค่ไหน
- ใกล้เงินแค่ไหน
- ใช้เวลาคนเก่งแค่ไหน

เลือกงานเดียวที่คะแนนสูงสุด

**2. Current messy flow**

เขียนแบบไม่สวยก็ได้:

ตอนนี้งานนี้เริ่มจากไหน ใครส่งอะไรให้ใคร ใช้ไฟล์ไหน ติดตรงไหน ต้องรอใคร approve

อย่าเพิ่งแก้

แค่เห็นความจริงก่อน

**3. AI-first flow**

ถามใหม่ว่า:

ถ้า AI เป็น default executor งานนี้ควรไหลอย่างไร?

เขียนเป็น:

- input ที่ AI ต้องได้
- prompt / instruction ที่ต้องใช้
- data source ที่ AI ต้องอ่าน
- output ที่ AI ต้องส่งกลับ
- จุดที่คนต้อง approve

**4. Measurement**

เลือก 3 ตัววัดพอ:

- เวลาที่ลดได้
- error ที่ลดได้
- conversion / response time / follow-up rate ที่ดีขึ้น

ถ้าวัดไม่ได้ อย่าเพิ่งเรียกว่า automation

เรียกว่า experiment ก่อน

## Tool / workflow / agent example

ตัวอย่างที่ practical มากจาก transcript คือการใช้ Skills + Connectors + Schedule

ไม่ต้องเริ่มจากระบบใหญ่

เริ่มจาก mini agent หนึ่งตัวก่อน

เช่น **Invoice Review Agent**

**Input**
- email จากคู่ค้า
- invoice attachment
- vendor name
- amount
- due date

**AI step**
- อ่าน invoice
- เช็คข้อมูลสำคัญ
- เทียบกับ PO หรือข้อมูลใน folder
- สรุปว่าจ่ายได้ / ต้องเช็ค / ข้อมูลขาด

**Human approval**
- finance หรือ owner ดูเฉพาะเคสที่ flag
- ไม่ต้องเปิดทุกไฟล์เอง

**Output**
- ตารางสรุปใน Excel / Airtable
- message ส่งให้บัญชี
- reminder ก่อน due date

นี่คือ AI ที่เริ่มเป็น operator

ไม่ใช่เพราะมันฉลาดที่สุด

แต่เพราะงานถูกออกแบบให้มันทำซ้ำได้

อีกตัวอย่างหนึ่งคือ content workflow

คุณมี transcript จาก workshop หรือ voice note

อย่าให้ AI แค่ “สรุปให้หน่อย”

ให้สร้าง Skill ที่ชัดว่า:

- ดึง theme
- หา business insight
- แตกเป็น newsletter
- แปลงเป็น LinkedIn post
- สร้าง carousel outline
- ส่งให้คน approve

นี่คือสิ่งที่ทำให้ทีมเล็กทำงานเหมือนทีมใหญ่

ไม่ใช่เพราะคนเก่งขึ้นทุกคน

แต่เพราะระบบช่วยให้มาตรฐานของงานซ้ำดีขึ้นทุกครั้ง

## Closing line

AI automation ไม่ได้เริ่มจาก tool

มันเริ่มจากเจ้าของธุรกิจยอมออกแบบงานให้ชัดพอที่ AI จะทำแทนได้

ระบบชนะ hero
