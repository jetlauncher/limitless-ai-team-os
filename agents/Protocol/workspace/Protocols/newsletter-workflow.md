# Protocol Newsletter Workflow

## Weekly operating flow

1. **Collect inputs**
   - Jet voice notes
   - Signal research summaries
   - high-performing posts
   - workshop/student questions
   - AI tool/workflow discoveries

2. **Choose the angle**
   Score each idea by:
   - usefulness for Thai business owners
   - urgency/timeliness
   - Jet authority fit
   - specificity
   - ability to create one concrete action

3. **Draft the issue**
   Include:
   - 5 subject lines
   - 2 preview texts
   - main Thai issue
   - operator move
   - tool/workflow example
   - CTA
   - repurposing pack

4. **Quality check**
   - Does it respect the reader's intelligence?
   - Is there a real business consequence?
   - Is there one thing they can do this week?
   - Does it sound like Jet, not a generic AI newsletter?
   - Are claims sourced if current/news-based?

5. **Approval gate**
   Send Kelly/Jet a concise summary:
   - recommended subject line
   - 3-bullet issue summary
   - file path / Beehiiv draft link if available
   - ask: approve, revise, or kill?

6. **Archive**
   - Drafts go in `Drafts/`
   - Final approved versions go in `Published/`
   - Metrics go in `Analytics/`

## Safety
Never publish, schedule, or email subscribers without explicit approval.
