# Run 01–02 Shape-First Carousel Lessons

Use this as session-specific guidance for future Jedi/Jet minimalist carousel runs.

## What Jet liked

- Contact sheet plus individual PNG delivery in Telegram is the expected preview workflow.
- Short praise like “Fire” / “This is fire” means the format is landing; keep moving, don’t oversell.
- “Build me the next one” should be treated as permission to autonomously pick another strong Plaud/Notion content idea and render a full next run.

## Copy patterns that worked

### Run 01 — 10-80-10
- Slide-before-last proof story should be concrete, not abstract.
- Strong version of the insurance agent story:
  - insurance/AIA sales leader
  - 25 years of experience
  - built 3 apps himself
  - those apps would have cost millions / large build cost if outsourced
  - reason: his 25 years of domain experience let him know the right problem better than AI or developers
- Jet explicitly wanted: “he built 3 apps that would have cost millions of baht by himself because of his 25 years of experience.” Preserve that causal chain.

### Run 02 — Brand Book / AI agency
- Business-owner math slide should be written in clean comparable units:
  - `ทีมกราฟิก 35,000 / เดือน`
  - `เครื่องมือ AI 5,000 / เดือน`
  - `ส่วนต่าง 30,000 / เดือน`
  - `หรือ 360,000 / ปี`
- Avoid “ราว” or mixed phrasing on math slides when exact comparison is the point.

## Visual lessons

- First three slides can be simple diagrams, but Run 02 made the geometry more concept-led:
  1. crossed-out old cost → AI block
  2. stacked old-way friction blocks
  3. Brand OS orbit / system model
- Slide 2 should not feel like a generic flowchart; make the shape express the trap/tension.
- Shapes must be visually obvious at contact-sheet size, not just subtle background decoration. If the user says they do not see shapes, the failure mode is usually: low opacity, too-small geometry, too much text covering the diagram. Fix by making slide 1–3 diagrams large, high-contrast cream/gold, and concept-specific, with text anchored below or beside the diagram.
- Slide 7 is usually the densest; QA it separately and simplify line breaks before delivery.

## Delivery

For drafts, send:
1. contact sheet
2. zip bundle
3. all individual PNGs when user asks or when likely useful

Keep final chat concise. Let the images carry the work.
