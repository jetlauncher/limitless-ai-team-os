## SETUP (0:45–2:00) — What is an AI OS?

ก่อนอื่น AI OS คืออะไร?

AI OS ไม่ใช่ app ตัวเดียว
ไม่ใช่ prompt ตัวเดียว
ไม่ใช่ automation ตัวเดียว

AI OS คือระบบที่ทำให้ AI ทำงานกับชีวิตหรือธุรกิจของคุณได้ต่อเนื่อง

มี 5 ส่วนหลัก:

1. **Context** — AI รู้ว่าคุณเป็นใคร ธุรกิจคืออะไร
2. **Files** — AI เข้าถึงไฟล์ที่เกี่ยวข้องได้
3. **Instructions** — AI รู้วิธีทำงานกับคุณ
4. **Workflows** — มีวิธีทำงานซ้ำ ๆ เช่น content, meeting, sales, research
5. **Memory** — มีที่เก็บสิ่งที่ทำไปแล้วและสิ่งที่ควรรู้ต่อ

ถ้าคุณไม่มี 5 อย่างนี้ AI จะเป็นแค่ chatbot

แต่ถ้าคุณมี 5 อย่างนี้ AI จะเริ่มเป็น operating system

และสำหรับนักเรียนของผม วิธีเริ่มที่ง่ายที่สุดคือ:

เปิด Claude Cowork หรือ Codex
แล้วให้มันทำงานใน folder ที่จัดระบบไว้แล้ว

ไม่ใช่คุยลอย ๆ ใน chat window

เพราะ chat ลอย ๆ มันไม่มี filesystem
ไม่มี memory structure
ไม่มี source of truth

แต่ folder ที่ดีทำให้ AI มีบ้าน

## PART 5 (10:00–12:00) — Claude.md / AGENTS.md / CODEX.md: instructions for the AI worker

ตอนนี้เรามาถึงไฟล์ที่สำคัญมากสำหรับ Claude Cowork และ Codex

ไฟล์นี้คือ instruction file

ถ้าคุณใช้ Claude ecosystem อาจใช้ `CLAUDE.md`
ถ้าคุณใช้ Codex หรือ agent tools อาจใช้ `AGENTS.md` หรือ `CODEX.md`

ผมแนะนำให้มีไว้ใน folder นี้เลย

ตัวอย่าง:

```markdown
# AI Operating Instructions

You are working inside my personal/business AI OS folder.

## First steps
1. Read `00_START_HERE.md`.
2. Read any relevant files before answering.
3. Use the folder structure. Do not create random files at root unless necessary.

## Work style
- Give the short answer first.
- Be practical and specific.
- When drafting content, match `01_PROFILE/BRAND_VOICE.md`.
- When making plans, break them into small next actions.

## File rules
- Save drafts to `09_OUTPUTS/DRAFTS/`.
- Save reports to `09_OUTPUTS/REPORTS/`.
- Update `02_GOALS/DECISIONS.md` when a major decision is made.
- Never store passwords, API keys, or private tokens in markdown files.

## Safety
- Ask before deleting files.
- Ask before sending emails, posting content, or spending money.
- If unsure, mark `[NEEDS HUMAN REVIEW]`.

## Common tasks
- Turn notes into content ideas.
- Turn content ideas into scripts.
- Summarize meetings.
- Create weekly reviews.
- Draft sales copy.
- Organize messy notes into the right folder.
```

นี่คือสิ่งที่ทำให้ AI ไม่ใช่แค่ตอบคำถาม

แต่มันรู้วิธีทำงานในระบบของคุณ

เหมือนพนักงานใหม่ที่มี employee handbook

## PART 6 (12:00–14:00) — How to use this inside Claude Cowork

ถ้าคุณใช้ Claude Cowork หรือ Claude Projects หรือ environment ที่ Claude เข้าถึง folder/project ได้

workflow ที่ผมแนะนำคือ:

**Step 1:** เปิด folder `My AI OS` ใน Claude Cowork

**Step 2:** ให้ Claude อ่าน `00_START_HERE.md`

Prompt แรกที่ใช้ได้เลย:

```text
Please inspect this AI OS folder. Start with 00_START_HERE.md, then read the profile and instruction files. Tell me what context is clear, what is missing, and what 5 files I should improve first.
```

**Step 3:** ให้มันช่วยเติม gaps

```text
Based on the current folder, ask me 10 questions that would make this AI OS more useful for my business. After I answer, update the relevant markdown files.
```

**Step 4:** ใช้มันทำงานจริง

ตัวอย่าง prompt:

```text
Read my business context, brand voice, and content idea bank. Create 10 YouTube video ideas for my target customer. Save the best 3 as draft outlines in 09_OUTPUTS/DRAFTS/.
```

หรือ:

```text
Read my meeting notes and customer pain points. Extract the top 5 repeated customer objections and suggest how I should update my offer page.
```

นี่คือ difference ระหว่าง chat ธรรมดากับ AI OS

Chat ธรรมดา: คุณต้อง copy context ทุกครั้ง

AI OS: context อยู่ใน folder แล้ว AI ไปอ่านเอง

## PART 7 (14:00–16:00) — How to use this inside Codex

ถ้าคุณใช้ Codex หรือ coding-agent style workflow

Codex จะเก่งมากถ้า folder มี instructions และ file structure ที่ชัด

ถึงแม้งานของคุณไม่ใช่ coding 100% ก็ยังใช้ Codex เป็น operating agent ได้

เพราะ Codex ทำงานกับ files ได้ดี

ตัวอย่าง prompt แรก:

```text
Inspect this folder as my AI OS. Read 00_START_HERE.md and 08_AI_INSTRUCTIONS/CODEX.md. Do not modify files yet. Return a folder audit: missing context, duplicate files, unclear source-of-truth files, and recommended next edits.
```

หลังจากนั้น:

```text
Create or update the minimum set of markdown files needed to make this AI OS usable for weekly content planning. Do not invent business facts. Use [NEEDS INPUT] where information is missing.
```

แล้วสำหรับ weekly workflow:

```text
Run my weekly review workflow. Read 02_GOALS/THIS_MONTH.md, 05_CONTENT/CONTENT_IDEAS.md, and 06_MEETINGS_NOTES/. Create a weekly review report in 09_OUTPUTS/REPORTS/ with wins, problems, next actions, and 3 content ideas.
```

สำหรับ content:

```text
Use my brand voice and customer pain points to draft a 7-minute YouTube script. Save it to 09_OUTPUTS/DRAFTS/. Include hook, sections, CTA, and 5 shorts cutdowns.
```

สำหรับ business:

```text
Read CURRENT_OFFERS.md and CUSTOMER_OBJECTIONS.md. Suggest 5 improvements to my offer positioning. Mark anything that needs proof as [NEEDS PROOF].
```

Codex จะทำงานดีขึ้นมากเมื่อมันไม่ต้องเดาว่าไฟล์ไหนอยู่ไหน

folder structure คือ prompt ที่ถาวรกว่า prompt ใน chat

## PART 8 (16:00–17:45) — The weekly AI OS routine

ถ้าคุณอยากให้ระบบนี้ใช้งานจริง ผมแนะนำ routine ง่าย ๆ อาทิตย์ละครั้ง

ทุกวันศุกร์หรือวันอาทิตย์ เปิด Claude/Codex แล้ว run prompt นี้:

```text
Run my weekly AI OS review.

1. Read 00_START_HERE.md.
2. Read 02_GOALS/90_DAY_GOALS.md and THIS_MONTH.md.
3. Review new files in 09_OUTPUTS/ and 06_MEETINGS_NOTES/.
4. Summarize what changed this week.
5. Identify 3 priorities for next week.
6. Identify missing context that would make you more useful.
7. Save the review to 09_OUTPUTS/REPORTS/YYYY-MM-DD-weekly-review.md.
```

นี่จะทำให้ AI OS ของคุณ improve ทุกสัปดาห์

ไม่ต้อง perfect ตั้งแต่วันแรก

ระบบนี้เหมือน gym

ใช้บ่อย ๆ มันจะแข็งแรงขึ้น

ถ้าเจอว่า AI ตอบผิด เพราะไม่รู้ข้อมูลบางอย่าง
ไม่ต้องโกรธ AI

ให้สร้างไฟล์หรือ update context

ทุกครั้งที่ AI พลาด ให้ถามว่า:

“ข้อมูลนี้ควรอยู่ในไฟล์ไหน เพื่อให้ครั้งหน้ามันไม่พลาด?”

นี่คือวิธีคิดแบบ system builder

## PART 9 (17:45–19:15) — What not to put in your AI OS

สำคัญมากครับ

อย่าเอาทุกอย่างใส่มั่ว ๆ

โดยเฉพาะ:

- passwords
- API keys
- bank details
- full customer private data
- private ID documents
- raw confidential contracts ถ้าไม่จำเป็น

ให้เขียนแค่ path หรือ note ว่า credential อยู่ที่ไหน

เช่น:

```markdown
Google token is stored locally in the secure config folder. Do not print or copy token contents.
```

ไม่ใช่เอา token มา paste ใน markdown

อีกอย่างคืออย่าให้ AI ทำ external side effects โดยไม่ขออนุมัติ

เช่น:

- ส่ง email
- post social media
- ลบไฟล์
- จ่ายเงิน
- deploy production
- message customer

สำหรับนักเรียน ผมแนะนำ rule ง่าย ๆ:

**AI draft ได้, analyze ได้, organize ได้, plan ได้ — แต่ action ภายนอกต้องให้มนุษย์ approve ก่อน**

นี่คือ safety boundary ที่ทำให้คุณใช้ AI ได้แรงขึ้นโดยไม่เสี่ยงเกินไป

## PART 10 (19:15–21:00) — Build this in 60 minutes

ถ้าคุณอยากเริ่มวันนี้ ผมให้ 60-minute setup plan:

**Minute 0–10:** สร้าง folder structure

สร้าง `My AI OS` แล้ว copy folder structure ที่ผมให้

**Minute 10–20:** เขียน `00_START_HERE.md`

ตอบว่า AI OS นี้คืออะไร และ AI ควรเริ่มจากไฟล์ไหน

**Minute 20–30:** เขียน profile files

`USER_PROFILE.md`, `BUSINESS_CONTEXT.md`, `BRAND_VOICE.md`

**Minute 30–40:** เขียน `CLAUDE.md` หรือ `AGENTS.md`

บอก AI วิธีทำงาน วิธี save outputs และ safety rules

**Minute 40–50:** ใส่ content/business files จริง 3–5 ไฟล์

เช่น offer, content ideas, customer pain points, meeting notes

# Student Template — Minimal Files

## `00_START_HERE.md`

```markdown
# My AI OS — Start Here

This folder is my personal/business AI operating system.

## AI should help me with
- [task 1]
- [task 2]
- [task 3]

## Rules
- Read relevant files before answering.
- Do not invent facts.
- Mark missing information as [NEEDS INPUT].
- Save drafts to 09_OUTPUTS/DRAFTS/.
- Save reports to 09_OUTPUTS/REPORTS/.
- Ask before deleting, sending, posting, buying, or deploying.

## Start with these files
- 01_PROFILE/USER_PROFILE.md
- 01_PROFILE/BUSINESS_CONTEXT.md
- 01_PROFILE/BRAND_VOICE.md
- 08_AI_INSTRUCTIONS/CLAUDE.md
```

## `01_PROFILE/USER_PROFILE.md`

```markdown
# User Profile

## Name

## Role

## Current focus

## Communication preference

## What I want AI to help with

## What AI should avoid
```

## `01_PROFILE/BUSINESS_CONTEXT.md`

```markdown

## Channels

## Key metrics
```

## `01_PROFILE/BRAND_VOICE.md`

```markdown
# Brand Voice

## Tone

## Words we use

## Words we avoid

## Examples I like

## Examples I dislike
```

## `08_AI_INSTRUCTIONS/CLAUDE.md` or `AGENTS.md`

```markdown
# AI Instructions

You are working inside my AI OS folder.

1. Read `00_START_HERE.md` first.
2. Read relevant context before answering.
3. Keep answers practical and specific.
4. Save outputs in the right folder.
5. Do not invent missing business facts.
6. Ask before external side effects.
7. Mark uncertain items as [NEEDS HUMAN REVIEW].
```
