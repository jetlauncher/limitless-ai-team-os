# Our Version of “100 hours of Hermes Agent lessons in 23 minutes”

Reference video: https://youtu.be/k5NhsF7t68M
Reference title: “100 hours of Hermes Agent lessons in 23 minutes” by Jack Roberts

## What the reference is doing

It is not really a tutorial about one feature. It is a fast founder/operator walkthrough that sells a bigger idea:

> Most people use Hermes like a chatbot. Power users turn it into a personal/team operating system.

Structure:
1. Hook: “You are leaving 90% of Hermes on the table.”
2. Memory/Soul.md as the base layer.
3. Obsidian/local files as the real brain.
4. Meeting/email integrations as daily context.
5. Background tasks + cron as automation.
6. Claude Code / coding agents as execution engine.
7. Goals/super-goals as persistent direction.
8. Model switching / specialist models.
9. Multi-agent “Pantheon” system.
10. Mission Control OS / dashboard.
11. GitHub backup and portability.
12. Key commands and local vs VPS.
13. Next-video bridge.

## Our angle

We should not copy Jack’s video. Our version should be grounded in Jet’s actual operating system and brand:

**Working title options**
- “I Built an AI Team OS on Hermes — Here’s the Real Setup”
- “From Chatbot to AI Team: My Hermes Agent Operating System”
- “The Hermes Setup Running My AI Business in Thailand”
- “100 Hours Building a Real AI Team OS with Hermes”

**Core promise**
> I’ll show the exact Hermes setup I use to run an AI business: memory, agents, dashboards, revenue alerts, content systems, local files, and team command center — not theory, a real working stack.

**Audience**
- Thai founders/operators learning AI agents.
- Limitless Club students who need a practical AI Team OS.
- Advanced agent users who want multi-agent operations, not toy demos.

**Tone**
- Founder/operator, premium, calm, credible.
- Less hype than the reference; more “real system, real examples, real safeguards.”
- English or Thai can work. For Thai audience, use Thai voiceover with English technical labels on screen.

## Proposed 22–25 minute chapter map

### 00:00 — Hook: Most people use AI agents wrong
Key line: “If you only chat with Hermes, you’re using maybe 10% of what it can do. The real unlock is turning it into your AI team OS.”

On screen:
- Fast montage of Telegram command, Discord command center, Obsidian memory, Mission Control dashboard, cron job list, agent roster.

### 01:00 — Memory is the foundation
Show:
- Hermes memory/user profile concept.
- Obsidian `Agents/Hermes/Memory/MEMORY.md` and Daily notes.
- Difference between temporary task progress and durable memory.

Jet-specific example:
- Kelly remembers brand preferences, credential paths, agent roles, revenue source of truth.

### 02:30 — Soul / personality files turn agents into teammates
Show:
- Kelly/Hermes persona concept.
- Profile-specific `SOUL.md` for agents like Signal, Blaze, Bolt, Qwen, Oracle.

Key point:
- You don’t just ask “an AI.” You define a role, boundaries, memory policy, and mission.

### 04:00 — Obsidian as the shared agent brain
Show:
- `~/Documents/Obsidian Vault/Agents/`
- Hermes daily notes, Shared Memory, individual agent folders.

Example:
- A video upload, token fix, cron setup, and handoff all become readable notes.

### 05:30 — Telegram and Discord as command centers
Show:
- Telegram for fast DM/alerts.
- Discord for team command center.
- Explain no-mention / thread workflow at a high level.

Key line:
- “The interface doesn’t have to be a terminal. Your AI team can live where your team already talks.”

### 07:00 — Cron jobs: agents that wake up on schedule
Show real examples:
- Google/YouTube OAuth token health check every 4 days.
- Revenue / email alerts.
- Daily content/research/dashboard automations.

Safety point:
- Jobs can be no-agent scripts for simple watchdogs, or full agent jobs for reasoning.

### 08:45 — Revenue and business monitoring
Show:
- Mission Control / Airtable revenue source of truth.
- Near-real-time sales alerts.
- Difference between Stripe immediate alerts and Airtable reporting truth.

Key point:
- Agents become useful when tied to business state, not just chat.

### 10:30 — Multi-agent roster: one AI is not enough
Show agent map:
- Kelly = ops/chief-of-staff.
- Signal = AI research / X radar.
- Blaze = content.
- Pixel = visuals.
- Kaijeaw = Thai content.
- Bolt = apps/sites.
- Oracle = idea surfacing.
- Qwen = local/private worker.
- Nova-NJJ = separate iMac/local-files instance.

Key line:
- “This is not a chatbot. It’s an org chart.”

### 12:30 — Specialist models and routing
Explain:
- Kelly/default on GPT-5.5 via OpenAI Codex.
- Claude Code via Claude Max for coding/writing when needed.
- Grok/xAI for X-native research/media.
- Local Qwen for private/local work.

Safety point:
- Route work by capability and cost; don’t burn API spend blindly.

### 14:00 — Claude Code / coding agents as execution layer
Show:
- How Hermes can delegate or coordinate coding workflows.
- Example: build dashboard, generate report, upload video, fix token check.

Key point:
- Hermes is the operator layer; coding agents are the construction crew.

### 15:30 — Local files are the moat
Show:
- Agents reading local files/projects/Obsidian instead of generic web answers.
- NJJ iMac profile idea: use files already on that machine for the team.

Key line:
- “The agent that knows your real files becomes 10x more useful than a generic chatbot.”

### 17:00 — Mission Control dashboard
Show:
- Mission Control site / health endpoint / dashboard visuals.
- What it should eventually display: revenue, students, agent health, cron health, content queue.

### 18:30 — Backups and portability
Show:
- Private GitHub or backup workflow concept.
- Why configs, skills, memory, scripts need backup.

### 19:30 — Top commands / habits
Mention:
- `hermes profile list`
- `hermes cron list`
- `hermes gateway status`
- `hermes skills list`
- `/background`, cron, skills, memory, session search.

Frame as habits, not a command dump.

### 21:00 — Local vs cloud/VPS
Our take:
- Local Mac is best for private files and personal/team OS.
- VPS is useful for always-on services, but needs security.
- Hybrid is ideal: local private brain + cloud dashboards/webhooks where needed.

### 22:30 — Close: build your AI Team OS
CTA:
- “Don’t ask what AI tool to use. Design your AI team: memory, roles, tools, dashboards, review gates.”
- For Limitless Club: “This is the direction of the AI Team OS we’re building.”
