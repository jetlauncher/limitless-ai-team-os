# Kelly / Hermes Memory Workflow

## On every meaningful task
1. Use built-in Hermes memory for compact durable facts when appropriate.
2. Use this Obsidian workspace for human-readable memory and artifacts.
3. Use `Daily/` for near-term context and handoffs.
4. Promote stable facts to `Memory/MEMORY.md`.
5. Promote repeatable processes to `Protocols/`.
6. If another agent needs the context, add it to `Agents/Shared Memory/Daily/2026-04-26.md` or another shared note.

## Do not store
- Secrets or raw tokens.
- Temporary task progress in durable memory.
- Huge raw dumps unless they are intentionally archived.

## Routing
- Kelly/Hermes: ops, automation, config, coordination.
- Blaze: content creation and creative systems.
- Signal: research and high-signal monitoring.
- OpenClaw: legacy Friday/OpenClaw workflows.
