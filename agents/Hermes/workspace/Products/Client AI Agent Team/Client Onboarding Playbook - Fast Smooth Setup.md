# Client Onboarding Playbook - Fast Smooth AI Agent Setup

## Promise

Make the client feel this is simple, premium, and safe:

> In 7 days, your first private AI employee is set up with its own cloud computer, company memory, approved tools, and a weekly operating rhythm.

## Onboarding principles

1. Do not ask the client to understand tools.
2. Ask for outcomes, bottlenecks, access, and approval rules.
3. Start with one useful agent, not a big abstract AI team.
4. Use draft-only and approval gates first.
5. Make progress visible with a shared tracker and weekly Loom.
6. Keep security language simple: isolated workspace, scoped access, no passwords in chat, approval before risky actions.

## Fast onboarding flow

### Phase 0 — Pre-sale qualification, 15 minutes

Goal: make sure they are a safe and profitable fit.

Ask:
- What role do you wish you could hire this month?
- Which task wastes the founder/team the most time every week?
- Which tools does the team live in daily?
- What data is sensitive or off-limits?
- Who approves emails, external messages, payments, deletes, or posts?

Green flags:
- repetitive browser/admin work
- founder inbox/calendar chaos
- weekly reporting pain
- CRM/project management gaps
- content/research ops
- non-regulated business data

Red flags:
- healthcare/finance/legal sensitive data
- wants fully autonomous payments/posts/deletes immediately
- no internal owner
- messy access/security expectations

### Phase 1 — Paid kickoff, 60 minutes

Output: one clear AI employee mission.

Agenda:
1. Confirm business outcome.
2. Pick the first agent role.
3. Map current workflow.
4. Define approval gates.
5. Choose tools to connect.
6. Define success metric.
7. Agree on 7-day setup plan.

Recommended first agent roles:
- Founder Executive Assistant
- Sales Follow-up Assistant
- Research Assistant
- Content Operations Assistant
- Weekly Reporting Assistant
- CRM Cleanup Assistant

### Phase 2 — Access collection, 20 minutes client time

Use a simple checklist. Never collect passwords in chat/docs.

Collect:
- approved email account or delegated access
- calendar access
- project board access
- CRM access if needed
- company docs/knowledge sources
- brand docs or SOPs
- owner contact for approvals

Rules:
- Prefer OAuth/scoped service accounts.
- If password is unavoidable, use a password manager share, never Telegram/Notion/Obsidian.
- Separate each client into its own workspace/vault/credential boundary.

### Phase 3 — Build private workspace, internal

Create:
- Orgo workspace/computer or equivalent isolated VM
- Client Obsidian vault/folder
- Agent config/profile
- Tool credentials with least privilege
- Request/approval board
- Monitoring/watchdog
- Daily/weekly log file

Suggested folder structure:

```text
Clients/{Client Name}/
├── 00 - Client Brief.md
├── 01 - Access and Boundaries.md
├── 02 - Agent Mission.md
├── 03 - Workflows.md
├── 04 - Approval Rules.md
├── 05 - Weekly Reports/
├── 06 - Logs/
└── 99 - Offboarding.md
```

### Phase 4 — First useful workflow, 1-2 days

Do not build everything. Ship one valuable workflow first.

Examples:
- Draft weekly founder briefing
- Summarize inbox and open loops
- Research 20 leads and create CRM draft
- Produce weekly sales/content report
- Draft customer follow-up emails for approval
- Turn meeting notes into tasks

Definition of done:
- output exists
- client can review it
- approval rules are respected
- no high-risk autonomous actions
- weekly rhythm is scheduled

### Phase 5 — Client handoff demo, 30 minutes

Show:
- where the AI employee lives
- what it can do now
- what it cannot do yet
- how approvals work
- where reports/logs appear
- how to request new tasks
- how to pause/offboard access

Demo script:

> Here is your AI employee's private computer and memory. It only has access to the tools you approved. For now, it drafts and reports — it does not send, delete, pay, or publish without approval. Every week you receive a short report and we improve the workflow based on what saved the most time.

### Phase 6 — First 14 days success rhythm

Day 1:
- kickoff and access

Day 2-3:
- workspace setup and first workflow

Day 4:
- internal QA

Day 5:
- client demo

Day 7:
- first useful report/output

Day 14:
- ROI review and next workflow proposal

## Client intake form

Use these questions:

1. Company name and website
2. Main point of contact
3. What outcome do you want from your first AI employee?
4. Which role should we start with?
5. What tasks are repetitive or painful every week?
6. Which tools do you use daily?
7. Which tools can we connect in week 1?
8. What data is sensitive or off-limits?
9. What actions require human approval?
10. Who approves outputs?
11. What does success look like after 14 days?
12. Preferred weekly report format: email, Loom, Notion, PDF, or Telegram/Slack

## Approval gates

Always require approval for:
- sending external emails/messages
- social publishing
- payments/refunds/invoices
- deleting data
- changing automations
- deployments
- accessing large private datasets
- client-facing recommendations with legal/financial/medical implications

Safe autonomous actions:
- read approved docs
- summarize
- draft
- categorize
- create internal notes
- prepare reports
- update non-critical internal trackers if approved

## Smoothness hacks

- Send a 5-minute pre-kickoff video explaining the process.
- Give clients one checklist, not five documents.
- Use defaults: first agent = Executive Assistant unless another role is obvious.
- Keep kickoff focused on one workflow only.
- Show something useful within 72 hours.
- Use Loom for trust: clients like seeing the AI employee actually working.
- Make every task request go through one channel/board.
- Over-communicate boundaries: draft-only first, approvals for external actions.

## Example first-week scope

Client: service business founder

AI employee: Founder Executive Assistant

Week 1 workflows:
- summarize inbox/open loops daily
- turn meeting notes into tasks
- draft follow-up emails for approval
- prepare Friday weekly briefing

Success metric:
- saves founder 3-5 hours/week
- no missed follow-ups
- one weekly report delivered every Friday

Approval rules:
- may draft emails
- may create internal tasks
- may not send emails
- may not delete or change CRM records without approval
