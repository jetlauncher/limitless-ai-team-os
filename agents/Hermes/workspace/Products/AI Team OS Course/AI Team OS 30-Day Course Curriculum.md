# AI Team OS — 30-Day Self-Paced Course Curriculum

Owner: Jedi Trinupab / Limitless Club  
Prepared by: Kelly / Hermes  
Format: Self-paced online course  
Student level: Nontechnical founders, business owners, creators, consultants, operators  
Time commitment: 15–30 minutes/day  
Core promise: Build your own AI team in 30 days.

---

## Course Positioning

### Working Title

**AI Team OS: Build Your Own AI Team in 30 Days**

### One-line Promise

In 30 days, spending just 15–30 minutes per day, students will build a working AI team that can research, write, organize, remember, and support daily business operations.

### Who It Is For

- Business owners who want AI leverage but do not know where to start
- Nontechnical founders who feel overwhelmed by tools
- Coaches, consultants, educators, and service providers
- Operators who want a personal AI chief of staff
- Creators who want research, content, and workflow support
- Thai SMEs that want practical AI implementation, not theory

### Who It Is Not For

- People who want to become AI engineers
- People looking for deep coding/programming training
- People who want fully autonomous AI to run their company without supervision
- People unwilling to create accounts, copy/paste keys, or follow setup steps

### Transformation

Before the course:

> “I use ChatGPT sometimes, but it feels random and disconnected.”

After the course:

> “I have my own AI team with roles, memory, workflows, Telegram access, and simple automations.”

---

## Student Outcome by Day 30

By the end of the course, a student should have:

1. A personal AI Chief of Staff
2. A working Telegram interface
3. A basic memory/workspace system
4. 2–4 specialized AI agents
5. Agent role/instruction files
6. A simple daily/weekly operating rhythm
7. At least one scheduled automation
8. A private repo or backup folder for their AI system
9. A basic API key/security checklist
10. A final “AI Team OS Demo” they can show or use in their business

---

## Course Structure

### Format

- 30 days
- 4 weeks
- 6 core lessons/week + 1 catch-up/reflection day
- 15–30 minutes per day
- Each lesson includes:
  - Short concept video
  - Step-by-step build task
  - Copy/paste template
  - Checkpoint
  - Optional upgrade

### Learning Style

This course should feel like:

> “Build alongside me.”

Not:

> “Watch theory for 20 hours.”

The student should make progress almost every lesson.

# Course Map

## Week 1 — Foundation: Your First AI Chief of Staff

**Goal:** Get one working AI assistant that has a role, instructions, and a communication interface.

### Day 1 — The AI Team OS Mindset

**Objective:** Understand the difference between using AI as a chatbot and building AI as an operating system.

**Lesson Topics:**

- Why most people use AI too randomly
- Tool vs teammate vs operating system
- What an AI team is
- The 4 building blocks:
  - roles
  - memory
  - workflows
  - automation

**Student Task:**

- Write down 3 business problems they want AI to help with.
- Choose their first AI role: Chief of Staff, Researcher, Content Assistant, or Sales Assistant.

**Deliverable:**

- AI Team Goal Sheet

**Template:**

```text
My business:
My biggest bottleneck:
My first AI teammate should help me with:
Daily task I want help with:
Weekly task I want help with:
```

---

### Day 2 — Accounts and Tools Setup

**Objective:** Create the required accounts and understand API keys without fear.

**Lesson Topics:**

- What is an API key?
- Why AI tools need keys
- Which accounts are required vs optional
- Security basics: never share keys publicly

**Required Accounts:**

- Telegram
- GitHub
- OpenRouter or OpenAI
- Optional: Notion
- Optional: Obsidian

**Student Task:**

- Create accounts.
- Create a safe folder or password manager note for API key references.

**Deliverable:**

- API Key Checklist started

**Checkpoint:**

- Student understands where keys come from and where not to paste them.

---

### Day 3 — Install the AI Agent Runtime

**Objective:** Install the core tool that will run the student’s AI assistant.

**Lesson Topics:**

- What Hermes Agent is
- CLI vs chat interface
- Why we need a local runtime
- Basic install flow

**Student Task:**

- Install Hermes or the selected course runtime.
- Run a health check.

**Deliverable:**

- Working local AI agent command

**Nontechnical Support Note:**

This lesson should include:

- Mac walkthrough
- Windows workaround or cloud alternative
- Common errors
- Copy/paste commands
- “If stuck, watch this troubleshooting video”

---

### Day 4 — Create Your Chief of Staff Agent

**Objective:** Give the first agent a clear role and identity.

**Lesson Topics:**

- Why role design matters
- What a SOUL/instruction file is
- Good vs bad assistant instructions
- Chief of Staff operating principles

**Student Task:**

- Create their first agent profile.
- Write or copy a Chief of Staff instruction file.

**Deliverable:**

- First AI Chief of Staff profile

**Template Sections:**

```text
Agent name:
Mission:
What this agent owns:
What this agent should never do:
Preferred tone:
Daily responsibilities:
Approval rules:
```

---

### Day 5 — Connect Telegram

**Objective:** Let the student talk to their AI Chief of Staff from Telegram.

**Lesson Topics:**

- Why Telegram is a useful control surface
- BotFather basics
- Creating a bot safely
- Adding the bot token to the right place

**Student Task:**

- Create Telegram bot with BotFather.
- Add token to local `.env` file.
- Start gateway.
- Send `/start` and a test message.

**Deliverable:**

- Working Telegram AI assistant

**Checkpoint Message:**

```text
Hey, I’m your AI Chief of Staff. What should we organize today?
```

---

### Day 6 — First Real Delegation

**Objective:** Use the agent for a real business task.

**Lesson Topics:**

- How to delegate clearly
- Difference between asking and assigning
- How to request outputs in a reusable format

**Student Task:**

Ask the Chief of Staff agent to create:

- a weekly schedule
- a task list
- a business bottleneck analysis
- or a daily briefing format

**Deliverable:**

- First useful output from their AI Chief of Staff

**Example Prompt:**

```text
Act as my AI Chief of Staff. Ask me 5 questions about my business, then create a simple weekly operating rhythm for me.
```

---

### Day 7 — Catch-up and Reflection

**Objective:** Make sure the foundation works before moving on.

**Student Task:**

- Confirm agent runs
- Confirm Telegram works
- Confirm they know where the config files are
- Write 3 tasks they want to automate later

**Deliverable:**

- Week 1 checkpoint completed
