# Never Run Out of AI Limits Again

## The founder cheat sheet for getting more output from Claude, ChatGPT, and your AI team

**The real problem is not the limit. It is the workflow.** Most people burn usage because they use the strongest model too early, keep one giant chat forever, re-explain context repeatedly, and make one AI tool do every job.

### 1. Plan cheap, execute expensive
Use a cheap/fast model first to clarify the task. Only use the strongest model after you have a clear brief.

**Prompt:** “Interview me about the goal, audience, constraints, examples, and desired output. Then create a one-page brief.”

### 2. Stop using one giant chat
Long chats force the model to re-read old context. Use projects/folders and fresh chats for fresh tasks.

**Prompt:** “Create a concise handoff prompt so I can restart this in a new chat without losing context.”

### 3. Give AI memory outside the chat
Create 3 simple files:
- Founder Profile — who you are, voice, preferences
- Business Context — products, offers, audience, proof, goals
- AI Operating Rules — format, tone, approval gates, what to avoid

### 4. Escalate models like a manager
- Cheap/local model: brainstorm, summarize, clean notes
- Strong model: draft, analyze, create options
- Best model: final strategy, high-stakes writing, complex reasoning

Ask: **“What is the cheapest model that can do this step well?”**

### 5. Split tools by job
Use Gamma for slides, Notion/Obsidian for memory, Perplexity/web agents for research, Claude Code/Codex/Bolt for building, Qwen/local models for private drafts.

### 6. Use low-effort mode by default
For summaries, formatting, extraction, and action items, ask for concise output.

**Prompt:** “Be concise. I am optimizing for low token usage. If this chat becomes inefficient, tell me to start a new one.”

### 7. Turn repeated prompts into systems
If you ask the same prompt 3 times, make it a template, project instruction, skill, or automation.

## The 5-question checklist before every AI task
1. What outcome do I want?  
2. What context does the AI need?  
3. Which model/tool is cheapest but good enough?  
4. Should this be a fresh chat?  
5. Should this become reusable?

**Bottom line:** Stop treating AI like a magic textbox. Treat it like an operating system: plan cheap, execute expensive, keep memory clean, route work to the right specialist, and reuse workflows.
