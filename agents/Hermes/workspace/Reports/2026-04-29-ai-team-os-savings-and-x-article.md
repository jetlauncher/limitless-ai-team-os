## 1. Executive Summary

Over the last build cycle, we created the foundation of a private, multi-agent AI operating system for Jet and Limitless Club.

This is not just “using ChatGPT.” It is a working AI team structure with named agents, separate responsibilities, persistent memory, Telegram interfaces, Obsidian workspaces, GitHub version control, scheduled jobs, and a private student-cloneable repository.

The system now acts like a lightweight digital organization:

- **Kelly / Hermes** — chief of staff, ops, dashboards, approvals, orchestration
- **Signal** — AI research and market intelligence
- **Blaze** — content generation and repurposing
- **Kaijeaw** — Thai voice and Thai content polish
- **Zegna** — taste, brands, gadgets, trend scouting
- **Bolt** — apps, websites, landing pages, build support
- **Qwen** — local/private model worker
- **Protocol** — newsletter/Beehiiv editor and operator

The important business result: a founder can now operate with the leverage of a small AI-enabled team without hiring the full team immediately.

This does **not** mean AI replaces all humans perfectly. It means the founder can delay, reduce, or reshape hiring by automating research, writing, documentation, daily monitoring, coordination, and draft production.

## 2. What We Built Together

### 2.1 Multi-agent structure

We designed a named AI team where each agent has a clear job:

| Agent | Role | Business Value |
|---|---|---|
| Kelly / Hermes | Chief of staff and operating layer | Keeps the system organized, coordinates agents, monitors revenue/tasks, handles scheduled rhythms |
| Signal | Research and intelligence | Finds useful AI/business information and turns it into operator-ready insight |
| Blaze | Content engine | Drafts posts, articles, carousels, hooks, and repurposed content |
| Kaijeaw | Thai voice | Makes content sound native, sharp, and suitable for Thai business owners |
| Zegna | Taste and scouting | Watches cool tools, products, gadgets, and brands |
| Bolt | Builder | Supports websites, apps, automation, and product experiments |
| Qwen | Local model worker | Gives a path toward private/local AI work through Ollama |
| Protocol | Newsletter operator | Owns newsletter planning, drafting, Beehiiv workflow, and publishing checklist |

### 2.2 Telegram agent access

We created/connected dedicated Telegram bots so the founder can talk to agents like a real team:

- Kelly / Hermes
- Signal
- Blaze
- Kaijeaw
- Zegna
- Bolt
- Qwen
- Protocol

This matters because the interface becomes operationally natural. Instead of opening separate tools, the founder can delegate through chat.

### 2.3 Obsidian memory workspace

We created a structured Obsidian workspace under:

```text
~/Documents/Obsidian Vault/Agents/
```

With folders for:

- Hermes / Kelly
- Signal
- Blaze
- Kaijeaw
- Zegna
- Bolt
- Qwen
- Protocol
- Shared Memory

This gives the system a human-readable operating memory, not just hidden chat history.

### 2.4 Protocol newsletter agent

We created a dedicated newsletter agent named **Protocol**.

Protocol owns:

- newsletter positioning
- weekly issue planning
- research intake
- draft writing
- Beehiiv draft workflow
- approval checklist
- issue archive
- analytics loop

Current newsletter positioning:

> “A weekly operating memo for Thai business owners learning how to run their company with AI.”

### 2.5 Private GitHub repo

We created a private GitHub repo:

[https://github.com/jetlauncher/limitless-ai-team-os](https://github.com/jetlauncher/limitless-ai-team-os)

The repo includes:

- English README
- Thai README
- English installation guide
- Thai installation guide
- API key checklist
- architecture docs
- operations manual
- agent SOUL files
- sanitized config examples
- `.env.example` templates
- export/update scripts
- secret-scan workflow

This turns Jet’s personal AI team into a future student-cloneable system.

### 2.6 Daily repo update automation

We created a scheduled job:

```text
daily-limitless-ai-team-os-repo-refresh
```

It runs daily at **03:30 Bangkok time**.

It:

1. refreshes the sanitized repo from the live agent system,
2. runs a secret scan,
3. commits changes only if safe,
4. pushes to GitHub,
5. alerts Jet with the result.

This means the repo becomes a living source of truth as the system grows.

## 5. Where AI Is Strong vs Where Humans Are Still Needed

### AI is already strong for:

- first drafts
- research synthesis
- summarization
- translation
- content repurposing
- documentation
- checklists
- scheduled monitoring
- repo updates
- structured operating memory
- lightweight technical scaffolding
- dashboard and workflow design

### Humans are still important for:

- final judgment
- taste
- leadership
- relationship-building
- sales conversations
- legal/accounting decisions
- brand-defining decisions
- sensitive customer communication
- strategic prioritization

The correct model is not “AI replaces the company.”

The correct model is:

> AI becomes the operating layer that lets a small human team move like a much larger one.
