# Jet Version Video Package — Hermes Agent as AI Employee

Source inspiration: https://youtu.be/RoBD7Lc-0MI?si=m9giESxUCsFt8Vdj  
Working title: **ผมใช้ AI Agent เป็นพนักงาน 24/7 ของธุรกิจ — วิธีเริ่มใช้ Hermes Agent สำหรับเจ้าของกิจการไทย**

## Recommended angle

Don’t make a generic “Hermes tutorial.” Make Jet’s version about **how Thai founders can turn AI from a chatbot into an operating system for the business**.

The original video is a broad installation + feature walkthrough. Jet’s version should be more premium, strategic, and founder-relevant:

- Less hype, more operator clarity.
- Less “cool AI tool,” more “business leverage system.”
- Thai-primary, with examples from SME/content/course/operator workflows.
- Position Hermes as the control layer for a Founder AI Operating System.

## 5 title options

1. **AI Agent ที่ทำงานแทนพนักงานได้ 24/7 — ผมใช้ Hermes Agent ยังไงในธุรกิจจริง**
2. **เลิกใช้ AI แค่ตอบแชท: วิธีสร้าง AI Operating System ให้ธุรกิจด้วย Hermes Agent**
3. **Hermes Agent คืออะไร? คู่มือเริ่มต้นสำหรับเจ้าของกิจการไทย**
4. **ถ้าคุณมี AI Employee 1 คนในธุรกิจ ควรให้มันทำอะไรบ้าง?**
5. **ผมจ่ายให้ AI Agent เพื่อทำงานที่เคยต้องใช้ทีม — นี่คือระบบหลังบ้านทั้งหมด**

Recommended: **#2** for authority + brand fit. Use #1 if you want a stronger YouTube CTR hook.

## Core positioning

**Thesis:**
AI ที่ทรงพลังที่สุดไม่ใช่ chatbot ที่ตอบเก่งกว่าเดิม แต่คือ agent ที่จำบริบทของเรา ทำงานแทนเราได้ และพัฒนาวิธีทำงานของตัวเองให้ดีขึ้นเรื่อย ๆ

**Market implication for Thai founders:**
The next advantage is not “who knows the most AI tools.” It is **who designs the clearest workflows for AI agents to run every day**.

## Suggested video structure — 30–40 minutes

### 0:00 Hook — The employee frame

**Hook script:**

> ถ้าวันนี้คุณมีพนักงานคนหนึ่งที่ทำงานได้ 24 ชั่วโมง ไม่ลืมสิ่งที่คุณเคยสอน ไม่ต้องรอให้คุณเปิดคอม และทุกครั้งที่ทำงานพลาด มันเรียนรู้เพื่อทำให้ดีขึ้นครั้งหน้า…
>
> คุณจะให้เขาทำงานอะไรในธุรกิจของคุณ?
>
> วันนี้ผมจะพาไปรู้จัก Hermes Agent — ไม่ใช่แค่ AI chatbot แต่เป็น AI employee ที่ผมมองว่าเจ้าของกิจการควรเริ่มเข้าใจตั้งแต่ตอนนี้

### 1:30 What Hermes Agent is — not chatbot, but operating layer

Key beats:
- Hermes คือ AI agent ที่อยู่บนเครื่อง/ระบบของเรา
- สั่งงานผ่าน Telegram/Discord/iMessage ได้
- ใช้ tools ทำงานจริง: อ่านไฟล์, เขียนเอกสาร, research, schedule job, generate content, browse, operate computer
- มี memory และ skills ที่พัฒนาได้
- ทำงานเชิง proactive ได้ผ่าน scheduled tasks / cron jobs

Thai phrasing:

> ChatGPT คือคนที่คุณเรียกมาถามคำถาม
>
> Hermes Agent ใกล้เคียงกับพนักงานที่คุณมอบหมายงาน แล้วให้เขาไปจัดการ workflow ต่อเอง

### 4:00 Why founders should care now

Founder examples:
- สรุปข่าวคู่แข่งทุกเช้า
- เขียน draft content จาก research ทุกวัน
- เช็คยอดขายหรือ dashboard แล้วเตือนเมื่อมี signal ผิดปกติ
- สรุป meeting / transcript เป็น action plan
- ทำ customer research จาก comments/reviews
- สร้าง prototype หรือ internal tool ตอนกลางคืน

Framework:

**From Prompt → Workflow → Operating System**

- Prompt: ถามทีละเรื่อง
- Workflow: ให้ AI ทำงานซ้ำได้ตามขั้นตอน
- Operating System: ให้ AI จำบริบท, ทำงานตามเวลา, ส่งงานให้ตรวจ, ปรับปรุงวิธีทำงานเอง

### 8:00 Hermes vs Claude Code / Codex / normal chatbots

Simple founder lens:
- ChatGPT / Claude: best for thinking, drafting, analysis
- Claude Code / Codex: best for serious software development
- Hermes: best for always-on business operations, personal assistant, content ops, research ops, admin ops, lightweight builds

Line:

> ถ้าผมจะ build software หนัก ๆ ผมยังใช้ Claude Code หรือ Codex
>
> แต่ถ้าผมอยากให้ AI เป็นผู้ช่วยประจำตัวที่รู้จักธุรกิจผม ทำงานซ้ำทุกวัน และคุยกับผมผ่าน Telegram — ผมใช้ Hermes

### 12:00 The first setup that actually matters — teach it your business

Instead of focusing only on installation, show the onboarding prompt:

```text
คุณคือ AI Chief of Staff ของผม
นี่คือธุรกิจของผม: [ธุรกิจ/offer]
ลูกค้าหลักคือ: [ICP]
รายได้มาจาก: [revenue streams]
เป้าหมาย 90 วันคือ: [goals]
งานที่ผมทำซ้ำทุกสัปดาห์คือ: [recurring work]
สิ่งที่ผมไม่อยากให้คุณทำเองโดยไม่ขออนุญาตคือ: [boundaries]
จำบริบทนี้ไว้ และช่วยเสนอ 10 workflows ที่คุณควรช่วยผมได้ทันที
```

Explain why this is powerful:
- AI ต้องรู้ context ก่อนถึงจะทำงานดี
- ถ้าไม่สอน business context มันจะเป็น generic assistant
- ถ้าสอนดี มันจะเริ่มเป็น operator ของธุรกิจ

### 16:00 5 workflows every Thai founder can build first

1. **Daily Market Intel**
   - ทุกเช้าหาข่าว AI/ธุรกิจที่เกี่ยวกับอุตสาหกรรมเรา
   - สรุป implication ไม่ใช่แค่ summary

2. **Content Engine**
   - ดึง research → สร้าง hooks → draft posts → รอ approval
   - ไม่ auto-post ถ้าไม่อนุมัติ

3. **Sales Signal Monitor**
   - ดูยอดขาย/leads/messages
   - เตือนเมื่อมี drop/spike
   - เสนอ action

4. **Meeting-to-Execution System**
   - transcript → decisions → tasks → follow-up messages

5. **Founder Second Brain**
   - จำสิ่งที่คุยไว้
   - ค้น session เก่า
   - เปลี่ยน idea ที่เคยพูดเป็น plan จริง

### 24:00 Dashboard / Mission Control angle

Make this Jet-specific:

> จุดที่ผมชอบมากคือ เราไม่จำเป็นต้องให้ AI อยู่แค่ในกล่องแชท
>
> เราสามารถสร้าง Mission Control ของตัวเองได้ เช่น content pipeline, approval queue, sales dashboard, research dashboard — แล้วให้ agent ไปทำงานข้างหลังระบบนั้น

Examples to show visually:
- approval dashboard for posts
- daily content queue
- campaign monitor
- research database
- “what should I do today?” operator panel

### 29:00 Safety and approval rules

Jet’s version should be more responsible than the original:

- Never give full autopilot to public posting or paid actions without approval
- Separate “draft” from “publish”
- Set clear boundaries: money, deletion, credentials, client data
- Use approvals for social posts, payments, destructive actions

Line:

> ผมไม่ได้อยากให้ AI ทำทุกอย่างเองโดยไม่คิด
>
> ผมอยากให้ AI ทำงาน 80% แรก แล้วให้มนุษย์ตรวจ 20% สำคัญสุด
>
> นี่คือ 10-80-10 ที่ผมใช้กับ AI workflows

### 34:00 Final takeaway

> คนที่ชนะในยุค AI ไม่ใช่คนที่มี tools เยอะที่สุด
>
> แต่คือคนที่ออกแบบงานให้ AI ทำซ้ำได้ ชัดเจน วัดผลได้ และปลอดภัย
>
> Hermes Agent เป็นหนึ่งในเครื่องมือที่ทำให้ภาพนี้เริ่มจับต้องได้จริง

CTA:

> ถ้าคุณอยากให้ผมทำ Part 2 ผมจะพา setup workflow จริง 3 ตัวสำหรับเจ้าของกิจการไทย: daily market intel, content engine, และ sales monitor

## What to avoid copying from the original

- Do not copy the exact “number one AI tool on planet Earth” hype style.
- Do not make the video only an installation tutorial.
- Do not use the same security framing; Jet’s version should emphasize approval gates and responsible autonomy.
- Do not over-focus on OpenClaw drama. Mention alternatives briefly, then move to founder use cases.

## Recommended production format

- Long-form YouTube: 30–40 minutes
- Language: Thai primary, with English terms where natural: AI Agent, workflow, Mission Control, approval, operating system
- Visuals:
  - Talking head intro
  - Screen recording of Hermes/Telegram/Mission Control
  - Simple framework slides: Prompt → Workflow → Operating System, 10-80-10, 5 starter workflows
- Follow-up Shorts: 3–5 clips from the frameworks
