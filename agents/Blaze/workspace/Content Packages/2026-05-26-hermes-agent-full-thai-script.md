ไม่ใช่แค่ตอบคำถาม
ไม่ใช่แค่ช่วยเขียน caption

คำถามคือ…คุณจะให้พนักงานคนนี้ทำงานอะไรในธุรกิจของคุณ?

## 1:45 — What Hermes Agent actually is

ก่อนอื่น Hermes Agent คืออะไร?

ให้คิดง่าย ๆ แบบนี้ครับ

ChatGPT คือคนที่คุณเรียกมาถามคำถาม

แต่ Hermes Agent ใกล้เคียงกับพนักงานที่คุณมอบหมายงาน แล้วให้เขาไปจัดการ workflow ต่อเอง

มันเป็น AI Agent ที่สามารถอยู่ในระบบของคุณ
เชื่อมกับ tools หลายอย่าง
คุยกับคุณผ่าน Telegram, Discord, iMessage หรือช่องทางที่คุณใช้อยู่แล้ว

และที่สำคัญคือมันไม่ได้รอแค่ให้คุณถามคำถาม

มันสามารถทำงานตามเวลาได้
สามารถจำบริบทได้
สามารถค้น session เก่าได้
สามารถใช้ tools ได้
สามารถสร้างไฟล์ แก้ไฟล์ research ข้อมูล ทำเอกสาร สร้าง dashboard หรือแม้แต่ช่วย operate computer ได้ในบางกรณี

ถ้าพูดในภาษาธุรกิจ

Hermes ไม่ใช่แค่ AI ที่ตอบคำถาม

มันคือ operating layer ระหว่างคุณกับงานดิจิทัลของคุณ

สมมติคุณบอกว่า

ทุกเช้า ช่วยดูข่าว AI ที่เกี่ยวกับธุรกิจ SME ไทย แล้วสรุปมาให้ว่าอะไรสำคัญ และผมควรเอาไปทำ content อะไร

หรือ

ทุกวันตอน 8 โมง ช่วยเตรียม draft content สำหรับ LinkedIn, X, Threads จาก research ล่าสุด แล้วใส่ไว้ใน approval queue ให้ผมตรวจ

หรือ

ทุกเย็น ช่วยเช็คยอดขาย leads และข้อความลูกค้า ถ้ามีอะไรผิดปกติให้แจ้งผม

สิ่งเหล่านี้ไม่ใช่แค่ prompt เดี่ยว ๆ

มันคือ workflow

และ Hermes ถูกออกแบบมาให้เหมาะกับ workflow แบบนี้มากกว่า chatbot ปกติ

## 4:30 — The big shift: Prompt → Workflow → Operating System

ถ้าผมต้องสรุปวิดีโอนี้เป็น framework เดียว

ผมจะใช้คำว่า

**Prompt → Workflow → Operating System**

ระดับแรกคือ Prompt

นี่คือสิ่งที่คนส่วนใหญ่ทำกับ AI ตอนนี้

เปิด AI ขึ้นมา แล้วถามว่า
ช่วยเขียนโพสต์ให้หน่อย
ช่วยสรุปบทความนี้ให้หน่อย
ช่วยคิดไอเดียให้หน่อย

มันมีประโยชน์มากนะครับ
แต่ข้อเสียคือทุกอย่างขึ้นอยู่กับการถามของเราในตอนนั้น

ถ้าวันนี้เราลืมถาม มันก็ไม่เกิด
ถ้าเราให้ context ไม่ดี ผลลัพธ์ก็ generic
ถ้าเราต้องทำซ้ำทุกวัน เราก็ยังต้องเป็นคนเริ่มทุกครั้ง

ระดับที่สองคือ Workflow

นี่คือการเริ่มบอก AI ว่า
นี่คือขั้นตอน
นี่คือ input
นี่คือ output
นี่คือ format
นี่คือเวลาที่ต้องทำ
นี่คือเกณฑ์คุณภาพ
นี่คือสิ่งที่ต้องส่งให้มนุษย์ตรวจ

ยกตัวอย่างเช่น content workflow

1. ดูข่าวและ signal ล่าสุด
2. เลือกเฉพาะเรื่องที่เกี่ยวกับ founder หรือ SME
3. สรุป market implication
4. เขียน hook 5 แบบ
5. สร้าง post draft
6. ใส่เข้า approval queue
7. รอมนุษย์ approve ก่อน post

นี่เริ่มไม่ใช่ prompt แล้ว
นี่คือระบบงาน

ระดับที่สามคือ Operating System

Operating System คือเมื่อ workflow เหล่านี้ไม่ได้อยู่แยกกัน
แต่มันเริ่มเชื่อมกัน

มี memory
มี schedule
มี approval
มี dashboard
มี reports
มี tooling
มี feedback loop

นี่คือจุดที่ AI เริ่มกลายเป็นระบบหลังบ้านของธุรกิจ

และผมคิดว่า Hermes Agent น่าสนใจ เพราะมันพาเราไปสู่ระดับที่สามได้ง่ายกว่า AI chat ปกติ

## 7:30 — Why Thai founders should care

คำถามคือ แล้วทำไมเจ้าของกิจการไทยต้องสนใจเรื่องนี้ตอนนี้?

เพราะในธุรกิจจริง งานที่กินเวลาของ founder จำนวนมากไม่ใช่งานที่ยากที่สุด

แต่เป็นงานที่ซ้ำที่สุด
งานที่กระจัดกระจายที่สุด
งานที่ต้องจำ context เยอะที่สุด
และงานที่ไม่มีใครในทีมเป็นเจ้าของชัดเจน

เช่น

- ต้องอ่านข่าวตลาด แต่ไม่มีเวลาอ่าน
- ต้องทำ content สม่ำเสมอ แต่ไม่มีระบบ research
- ต้อง follow up ลูกค้า แต่ข้อมูลอยู่หลายที่
- ต้องสรุป meeting แต่สุดท้าย action item หาย
- ต้องดู dashboard แต่ไม่มีใครเตือนเมื่อมี signal สำคัญ
- ต้องสร้างเอกสารซ้ำ ๆ แต่ทุกครั้งเริ่มจากศูนย์

AI Agent ไม่ได้แก้ทุกอย่างทันที

แต่ถ้าออกแบบดี มันสามารถรับงานหลังบ้านเหล่านี้ไปได้เยอะมาก

และสำหรับธุรกิจไทย โดยเฉพาะ SME หรือทีมเล็ก

นี่คือ leverage ที่ใหญ่มาก

เพราะคุณอาจยังไม่พร้อมจ้างทีม data analyst
ไม่พร้อมจ้าง content researcher
ไม่พร้อมจ้าง operations assistant
ไม่พร้อมจ้าง automation engineer

แต่คุณสามารถเริ่มออกแบบ AI Agent ให้ทำหน้าที่บางส่วนของ role เหล่านี้ได้

ไม่ใช่แทนคน 100%

แต่ทำงาน 80% แรกให้เสร็จ
แล้วให้มนุษย์ใช้ judgment ใน 20% สำคัญสุด

นี่คือหลัก 10-80-10 ที่ผมใช้บ่อยมาก

10% แรก มนุษย์กำหนด direction
80% กลาง ให้ AI ทำงานหนัก
10% สุดท้าย มนุษย์ตรวจ decision สำคัญ

## 10:30 — Hermes vs ChatGPT, Claude Code, Codex

หลายคนอาจถามว่า แล้วต่างจาก ChatGPT ยังไง?

หรือถ้าใช้ Claude Code / Codex อยู่แล้ว ยังต้องสนใจ Hermes ไหม?

ผมแยกแบบนี้ครับ

ChatGPT หรือ Claude ปกติ เหมาะมากสำหรับการคิด วิเคราะห์ เขียน สรุป วางแผน

ถ้าคุณต้องการคุยกับ AI เพื่อ refine idea หรือเขียนอะไรบางอย่าง มันดีมาก

Claude Code หรือ Codex เหมาะมากสำหรับ software development

ถ้าคุณจะ build app จริงจัง แก้ codebase ใหญ่ ๆ ทำ testing ทำ pull request ทำ refactor ผมยังมองว่าเครื่องมือ coding เหล่านี้เหมาะกว่า

แต่ Hermes เหมาะกับอีกประเภทหนึ่ง

คือ always-on business operations

งานที่คุณอยากให้ AI อยู่กับคุณทั้งวัน
คุยผ่าน Telegram
จำบริบทของคุณ
ทำงานตามเวลา
สร้าง reports
จัด queue
ช่วย research
ช่วยเป็น chief of staff
ช่วยทำ admin
ช่วย build internal tools เล็ก ๆ

ถ้าพูดง่าย ๆ

ChatGPT คือที่ปรึกษาที่คุณเรียกมาคุย
Claude Code / Codex คือ engineer ที่ช่วย build software
Hermes คือ AI operator ที่อยู่กับธุรกิจคุณทุกวัน

และนี่คือเหตุผลที่ผมสนใจมัน

ไม่ใช่เพราะมันแทนทุก tool
แต่เพราะมันเชื่อมช่องว่างระหว่าง AI กับ workflow จริงของ founder

## 14:00 — The first setup that matters: Teach it your business

สิ่งแรกที่คนส่วนใหญ่ทำผิดเวลาใช้ AI Agent คือรีบไปถามว่า tool นี้ทำอะไรได้บ้าง

แต่คำถามที่ดีกว่าคือ

เราอยากให้มันเป็นพนักงานแบบไหนในธุรกิจเรา?

เพราะถ้าคุณไม่สอน context มันก็จะเป็น generic assistant

เหมือนจ้างพนักงานใหม่ แต่ไม่บอกว่าธุรกิจทำอะไร ลูกค้าคือใคร เป้าหมายคืออะไร ขอบเขตคืออะไร แล้วคาดหวังให้เขาทำงานดีทันที

เป็นไปไม่ได้

ดังนั้น prompt แรก ๆ ที่ผมแนะนำให้ใช้กับ AI Agent คือ onboarding prompt

ตัวอย่างเช่น

คุณคือ AI Chief of Staff ของผม

นี่คือธุรกิจของผม: [อธิบายธุรกิจ]
ลูกค้าหลักคือ: [ICP]
รายได้มาจาก: [revenue streams]
เป้าหมาย 90 วันคือ: [goals]
งานที่ผมทำซ้ำทุกสัปดาห์คือ: [recurring work]
สิ่งที่ผมไม่อยากให้คุณทำเองโดยไม่ขออนุญาตคือ: [boundaries]

จำบริบทนี้ไว้ และช่วยเสนอ 10 workflows ที่คุณควรช่วยผมได้ทันที

สิ่งสำคัญคือไม่ใช่แค่บอกข้อมูล

แต่ต้องบอก role, goal, recurring work และ boundaries

เพราะ AI Agent ที่ดีต้องรู้สองอย่างพร้อมกัน

หนึ่ง มันควรช่วยอะไร
สอง มันไม่ควรทำอะไรเอง

## 18:00 — Five starter workflows for founders

ทีนี้ถ้าเริ่มใช้จริง ผมแนะนำ 5 workflows แรกที่เจ้าของกิจการควรลอง

### Workflow 1: Daily Market Intel

ทุกเช้าให้ agent ดูข่าว ตลาด คู่แข่ง หรือ industry ที่เกี่ยวข้องกับธุรกิจเรา

แต่ไม่ใช่แค่สรุปข่าว

ให้มันตอบ 3 อย่าง

1. เกิดอะไรขึ้น
2. ทำไมเจ้าของกิจการควรสนใจ
3. เราควรทำ content, offer, product หรือ action อะไรจาก signal นี้

นี่คือความต่างระหว่าง summary กับ business intelligence

AI ที่ดีไม่ใช่แค่บอกว่าเกิดอะไรขึ้น
แต่ช่วยแปลว่าแล้วเราควรทำอะไรต่อ

### Workflow 2: Content Engine

ถ้าคุณต้องทำ content สม่ำเสมอ อย่าเริ่มจากหน้าเปล่า

ให้ agent ทำงานแบบนี้

research ล่าสุด → extract angle → เขียน hook → สร้าง draft → ใส่ queue → รอ approval

โดยเฉพาะถ้าคุณเป็น founder ที่มีความคิดเยอะ แต่ไม่มีเวลานั่งเขียนทุกวัน

AI Agent ควรเป็น content operator ให้คุณ

ไม่ใช่แค่ copywriter

มันควรช่วยเลือก angle
ช่วยจำ voice
ช่วยเสนอ structure
ช่วยทำให้เนื้อหาต่อเนื่องกับ positioning ของคุณ

### Workflow 3: Sales Signal Monitor

ถ้าธุรกิจคุณมี leads, sales, messages, bookings, payments

คุณไม่ควรรอให้ปัญหาใหญ่ก่อนค่อยดู dashboard

ให้ agent ช่วย monitor signal

เช่น

ยอดขายวันนี้ต่ำกว่าค่าเฉลี่ยไหม
มีลูกค้าทักมาแต่ยังไม่ได้ตอบไหม
มีคนสนใจ offer แต่ยังไม่ได้ follow up ไหม
มี campaign ไหน performance ตกไหม

AI Agent ที่ดีควรบอกคุณว่า

นี่คือสิ่งที่เปลี่ยนไป
นี่คือความเสี่ยง
นี่คือ action ที่ควรทำ

### Workflow 4: Meeting-to-Execution

หลายองค์กรไม่ได้ขาด idea
แต่ขาด execution หลัง meeting

ให้ agent รับ transcript หรือ notes แล้วเปลี่ยนเป็น

- decision ที่ตกลงกันแล้ว
- owner ของแต่ละ task
- deadline
- follow-up message
- risks
- next meeting agenda

นี่คือ workflow ที่ง่าย แต่ impact สูงมาก

เพราะมันลดการลืม ลดการตีความผิด และทำให้ meeting กลายเป็น execution จริง

### Workflow 5: Founder Second Brain

อันนี้สำคัญมาก

Founder มี idea เยอะมาก
แต่ idea ส่วนใหญ่หายไป เพราะไม่ได้ถูก capture และ organize

AI Agent ที่มี session recall และ memory ช่วยให้คุณถามกลับได้ว่า

เดือนที่แล้วเราคุยเรื่อง offer อะไรไว้บ้าง
สัปดาห์ก่อนมี content idea ไหนที่ยังไม่ได้ทำ
เราเคยวาง plan launch อะไรไว้
มี insight ไหนที่ยังไม่ได้ execute

นี่คือ second brain ที่ไม่ได้แค่เก็บข้อมูล
แต่ช่วยดึงกลับมาใช้ตอนที่เหมาะสม

## 26:00 — Mission Control: AI should not live only in chat

อีก concept หนึ่งที่ผมชอบมากคือ Mission Control

## 30:30 — Safety: Don’t confuse autonomy with no approval

แต่เรื่องหนึ่งที่ต้องพูดชัด ๆ คือ safety

ผมไม่ได้เชื่อว่าเราควรให้ AI ทำทุกอย่างเองแบบไม่มี approval

โดยเฉพาะงานที่เกี่ยวกับ

- public posting
- paid ads
- money movement
- deleting files
- customer data
- legal or contractual decisions
- sensitive credentials

AI Agent ที่ดีไม่ใช่ agent ที่ไม่มีเบรก

แต่คือ agent ที่ทำงานส่วนใหญ่ให้เรา แล้วรู้ว่าเมื่อไหร่ต้องหยุดเพื่อขอ approval

นี่คือเหตุผลที่ผมชอบ 10-80-10

10% แรก มนุษย์กำหนดทิศทาง

เราบอก goal, context, rules, boundaries

80% กลาง AI ทำงานหนัก

research, draft, summarize, organize, build, compare, prepare

10% สุดท้าย มนุษย์ตรวจ decision สำคัญ

โพสต์นี้ควร publish ไหม
budget นี้ควรใช้ไหม
email นี้ควรส่งไหม
ลูกค้ารายนี้ควรตอบแบบไหน

AI ไม่ได้มาแทน judgment ของ founder

มันมาเอางานถึก งานซ้ำ งานกระจัดกระจาย ออกไปจากหัว founder

เพื่อให้ founder มีเวลามากขึ้นสำหรับ judgment ที่สำคัญจริง ๆ
