## LinkedIn posts scheduled

### 2026-04-28 10:00 BKK — LinkedIn — Managing agents becomes the real AI problem
- Angle: Google/Gemini Enterprise Agent Platform + agent governance

```text
The next AI bottleneck will not be access to models.

It will be managing the work those models are allowed to do.

We are moving from:

“Which AI tool should we use?”

to:

“How do we govern hundreds or thousands of AI-assisted workflows?”

For founders, this changes the job.

The advantage is no longer simply giving everyone ChatGPT.

The advantage is designing an AI operating layer:

• which workflows AI can touch
• which data it can access
• which outputs need human review
• which actions require approval
• which metrics prove the system is creating value

AI adoption is becoming less about experimentation and more about management design.

Before buying another tool, ask:

Can the business actually manage the agents it wants to deploy?
```

### 2026-04-29 10:00 BKK — LinkedIn — Production AI needs interfaces, not demos
- Angle: Claude MCP production systems

```text
Most AI demos fail when they meet production systems.

Not because the model is weak.

Because the business has not designed the interface between AI and work.

This is why MCP-style thinking matters.

The question is no longer:

“Can the model answer?”

The better question is:

“What tools, context, permissions, and review loops does the agent need to do useful work safely?”

For operators, this is the practical shift:

• prompts are not enough
• APIs are not always enough
• browser use is not always safe
• context needs structure
• tools need boundaries
• humans still need final judgment

Production AI is not a chatbot sitting beside the business.

It is an interface layer connected to the business.

Founders who understand this will build systems.
Everyone else will keep collecting demos.
```

### 2026-04-30 10:00 BKK — LinkedIn — Memory changes the agent equation
- Angle: Claude Managed Agents memory beta

```text
AI agents become much more useful when they stop starting from zero.

Memory changes the equation.

A memoryless assistant can help with a task.

A well-designed agent with memory can improve how a business works over time.

But this creates a new leadership question:

What should the agent remember?

Not everything deserves to become memory.

Good memory:

• captures stable preferences
• preserves useful context
• improves future decisions
• reduces repeated explanation
• compounds operational knowledge

Bad memory:

• stores noise
• preserves outdated assumptions
• creates privacy risk
• makes systems harder to audit
• turns temporary context into permanent bias

The business implication is simple:

AI memory is not just a feature.
It is a knowledge management decision.

Founders should treat it like infrastructure, not convenience.
```

### 2026-05-01 10:00 BKK — LinkedIn — Test agents before they touch real tools
- Angle: AWS ToolSimulator / safe agent testing

```text
The more tools an AI agent can use, the more dangerous a weak workflow becomes.

A chatbot giving a bad answer is annoying.

An agent taking the wrong action in a CRM, finance system, email inbox, or customer database is a business risk.

This is why agent testing matters.

Before an AI workflow touches live systems, founders should ask:

• What can this agent access?
• What actions can it take?
• What edge cases have we tested?
• What happens if the user gives unclear instructions?
• What requires human approval?
• How do we know the workflow is safe?

The future of AI operations will need more than prompts and enthusiasm.

It will need simulation, sandboxing, permissions, audit logs, and review loops.

If you would test a junior employee before giving them admin access, you should test an AI agent before giving it tools.
```

### 2026-05-02 10:00 BKK — LinkedIn — Computer use is not magic, it is workflow design
- Angle: Codex/browser/computer-use expansion

```text
Computer-use agents are becoming more capable.

They can browse, click, test flows, read files, capture screenshots, and iterate through tasks.

That sounds like magic.

But for businesses, the real question is more practical:

What work should they actually do?

The companies that benefit most will not be the ones that simply say, “Let the agent use the computer.”

They will be the ones that define:

• the task boundary
• the success condition
• the source of truth
• the review step
• the escalation path
• the rollback plan

Computer use makes AI feel more autonomous.

But autonomy without workflow design creates risk.

The founder lesson:

Do not automate vague work.
Design clear work, then give AI the right tools to execute it.
```
