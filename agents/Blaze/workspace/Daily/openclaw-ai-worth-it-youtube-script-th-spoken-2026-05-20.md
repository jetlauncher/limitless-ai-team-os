## Section 1: ต้นทุนจริงของการจ้างคน

ขอพูดตรงๆ เรื่องคน

คนดี

ผมทำงานกับคนที่มีความสามารถจริง ซื่อสัตย์ ขยัน และทำให้ธุรกิจดีขึ้น

เรื่องนั้นจริง

แต่อีกอย่างที่จริงเหมือนกัน: คนซับซ้อน

และผมพูดในฐานะคนที่ยอมรับว่าตัวเองก็เป็น asshole อยู่บ้าง

เราทุกคนเอาชีวิตทั้งหมดไปทำงานด้วยกัน

พนักงานของคุณลูกป่วยวันอังคาร

คุณเองก็มีสัปดาห์แย่ๆ ในเดือนพฤศจิกายน

มีการชนกันทางนิสัยที่ไม่มีใครอยากพูดถึง

มีคนได้ offer ดีกว่าแล้วลาออกสามอาทิตย์ก่อน launch

ไม่มีใครผิด มันแค่เป็นอย่างนั้น เพราะเราเป็นคน

เรามีอารมณ์ มีปัญหาสุขภาพ มีวิกฤตครอบครัวที่โผล่มาโดยไม่มีสัญญาณ

การจัดการทั้งหมดนั้นคือต้นทุนจริง

Founder ส่วนใหญ่ไม่เคยใส่มันลงใน spreadsheet

ลองนับดูว่าการจ้างคนหนึ่งคนมีอะไรบ้าง:

ประกาศรับสมัคร, คัดเลือก, สัมภาษณ์ – ผมนั่งสัมภาษณ์มาสักระหว่าง 500 ถึง 1,000 ครั้งตลอดอาชีพ แต่ละครั้งคือเวลา

Offer, Onboarding, ช่วง ramp ที่ยังไม่ได้ output เต็มที่

การ manage ต่อเนื่อง: check-in, feedback, คุยเรื่อง performance, ความเข้าใจคลาดเคลื่อน

และบางทีก็ exit

ยังไม่นับเงินเดือน ประกันสังคม อุปกรณ์ training และ overhead ที่ซ่อนอยู่

เทียบกับ AI agent

ต้นทุนคืออะไร:

Tokens, เครื่องสำหรับรัน, เวลาในการตั้ง system ให้ดี

แค่นั้น

ไม่มีการคุยเรื่องลาป่วย

ไม่มี notice สองอาทิตย์ในเวลาที่แย่ที่สุด

ไม่มีการเมืองในทีม

ไม่มีปัญหาแรงจูงใจตอนเช้าวันจันทร์

ถ้า agent หยุดทำงาน คุณแก้ prompt ปรับ workflow หรือเติม API credits

ความล้มเหลวของมันเข้าใจง่ายกว่ามาก

ผมไม่ได้บอกว่า AI ดีกว่าคนในทุกเรื่อง

แต่การเปรียบต้นทุนที่ถูกต้องไม่ใช่: "AI subscription กับไม่มี AI subscription"

เทียบที่ถูกต้องคือ: "AI agent กับทุกอย่างที่มาพร้อมการจ้างคน"

นั่นคือการสนทนาคนละเรื่องเลย

## Section 2: AI Agent ทำอะไรได้ดีจริง

หลังจากรัน OpenClaw มา 5 เดือน นี่คือสิ่งที่ใช้ได้ผลจริง

**อย่างแรก: งานซ้ำๆ ที่ volume สูง**

Research, Draft, Summarize, Process ข้อมูล

เอา input ที่รกๆ มาจัดเป็น output ที่มีโครงสร้าง

ถ้างานของคุณมี pattern agent จัดการได้ดี

ไม่บ่น ไม่เบื่อ ไม่ทำงานแย่ลงบ่ายวันศุกร์เทียบกับเช้าวันจันทร์

Consistency คือสิ่งที่คนประเมินต่ำไป

คนเก่งทำงานได้ดีส่วนใหญ่ของเวลา

Agent ทำแบบเดียวกันทุกครั้ง ถ้าออกแบบ system ได้ถูกต้อง

**อย่างที่สอง: ความเร็ว**

งานที่เคยใช้เวลาหนึ่งวันอาจเหลือชั่วโมงเดียว

งานที่เคยใช้เวลาชั่วโมงอาจเหลือสิบนาที

แล้วมันทบกัน

ถ้าธุรกิจของคุณวิ่งด้วย content, research, analysis, operations หรือ customer communication การบีบเวลานั้นคือ leverage โดยตรง

**อย่างที่สาม: ความพร้อมใช้งาน**

ไม่มี timezone ไม่มี out-of-office

คุณรัน process ตีสองก็ได้ มันไม่สน

**อย่างที่สี่: งานขนาน**

คนหนึ่งคนทำงานหนึ่งอย่างในเวลาหนึ่ง

Agent รัน thread หลายสายพร้อมกันได้

สำหรับทีมเล็ก นั่นเท่ากับการเพิ่มกำลังคนโดยไม่เพิ่มความซับซ้อนของการ manage

ตรงนี้แหละที่ AI น่าสนใจสำหรับ founder

ไม่ใช่เพราะมันเขียน caption น่ารัก

แต่เพราะมันให้ทีมเล็กทำงานได้เหมือนทีมใหญ่

## Section 3: สิ่งที่ AI Agent ทำไม่ได้

ผมไม่อยาก oversell เพราะการ oversell คือเหตุผลที่คนผิดหวัง

Agent ไม่ใช่เวทมนตร์

มันต้องตั้ง setup ให้ดี

Input คลุมเครือ: output ขยะ

คุณภาพของสิ่งที่ได้ผูกกับคุณภาพของการออกแบบงานโดยตรง

งาน setup นั้นคืองานจริง ต้องใช้เวลา ต้องทำซ้ำหลายรอบ

คุณต้อง define role, define context, define ว่า output ที่ดีหน้าตาเป็นยังไง

คุณต้อง review output แล้วปรับปรุง system

AI ไม่ได้แทน management

มันเปลี่ยนความหมายของ management

คุณไม่ได้แค่ manage คน แต่ manage system

Agent ยังแทน judgment ไม่ได้

คนอาวุโสที่เห็นอุตสาหกรรมของคุณมาสิบปีมี pattern recognition ที่ model ยังตามไม่ทัน

ความสัมพันธ์กับลูกค้า สัญชาตญาณเชิงกลยุทธ์ การอ่านห้อง taste จังหวะเวลา การตัดสินใจสุดท้าย

สิ่งเหล่านั้นยังเป็นเรื่องของคน

จุดประสงค์ไม่ใช่การเอาคนออกจากธุรกิจ

จุดประสงค์คือหยุดเสียความสนใจของคนไปกับงานที่เครื่องทำได้ดี

Agent ปลดปล่อยเวลาให้คนของคุณไปใช้กับ judgment ความสัมพันธ์ และความคิดสร้างสรรค์มากขึ้น

และใช่ agent พัง

Process ที่ตั้งไว้เดือนหนึ่งอาจต้องสร้างใหม่เดือนสาม เพราะ model เปลี่ยน tool เปลี่ยน หรือธุรกิจของคุณเปลี่ยน

การ maintain นั้นไม่ฟรี ต้องมีคนรับผิดชอบ

ถ้าไม่มีใครดูแล system: system จะเน่า

## Section 4: ประเมินยังไงว่า AI คุ้มกับธุรกิจของคุณ

นี่คือ framework ที่ผมใช้

### 1. Map งานซ้ำๆ ของคุณ

ทีมคุณทำงานอะไรทุกสัปดาห์:

Research, Writing, Summarize, Data entry, Draft ตอบลูกค้า, Internal report, Repurpose content, ติดตาม sales, จัดเอกสาร

List มันออกมา แล้วประเมินชั่วโมงต่อสัปดาห์

นั่นคือเป้าหมายของคุณ

อย่าเริ่มจากงานเชิงกลยุทธ์ที่สำคัญที่สุดในธุรกิจ

เริ่มจากงานซ้ำที่กินเวลาอยู่แล้ว

### 2. ใส่ต้นทุนจริงลงในชั่วโมงพวกนั้น

อย่านับแค่เงินเดือน

นับเวลาที่คุณเองใช้ manage คนที่ทำงานนั้น

นับ overhead

นับ error ที่คุณต้องจับและแก้

นับการสะดุดตอนมีคนลา

นับ meeting ที่ต้องมีเพื่อให้ทุกอย่างเดินต่อ

Founder ส่วนใหญ่นับน้อยไป

มองแค่เงินเดือนแล้วคิดว่านั่นคือต้นทุน

ไม่ใช่

ต้นทุนจริงคือ: เงินเดือน บวก management drag บวกความไม่ consistent บวกความเสี่ยงที่คนจะลาออก

### 3. เทียบกับต้นทุน setup AI จริง

เครื่อง, API credits, Software, เวลาในการสร้างและ maintain workflow

สำหรับ founder ส่วนใหญ่ ตัวเลขนี้เล็กกว่าที่คาดไว้

ไม่ใช่ศูนย์ แต่เล็กกว่า

และที่สำคัญกว่าคือมัน scale ต่างกัน

การจ้างคนเพิ่มต้นทุนคงที่และความซับซ้อนของการ manage

AI workflow เพิ่มต้นทุนตาม usage และความซับซ้อนของ system

สองอย่างนั้นไม่ใช่สิ่งเดียวกัน

### 4. ทดสอบก่อนตัดสินใจ

อย่าซื้อเครื่อง 200,000 บาทเพื่อทดสอบว่า AI เหมาะกับคุณไหม

เริ่มเล็กกว่า

เลือก workflow หนึ่งอัน รัน 30 วัน วัดผล:

ประหยัดเวลาได้เท่าไหร่?

สร้าง error กี่อัน?

ต้อง review มากแค่ไหน?

มันทำให้งานเร็วขึ้น ดีขึ้น หรือแค่ซับซ้อนขึ้น?

เครื่องคือการตัดสินใจเรื่อง scale ไม่ใช่การตัดสินใจเริ่มต้น

Mac Studio 100,000 บาท คือการตัดสินใจเรื่อง compute หลังจากรู้แล้วว่า workflow มีอยู่จริง

M5 Max 200,000 บาท อยู่ใน category เดียวกัน

ไม่ใช่ "ซื้อเครื่องแล้วหวังว่า AI จะทำงานได้"

แต่คือ: "workflow พิสูจน์ตัวเองแล้ว ดังนั้น compute เพิ่มมันสมเหตุสมผล"

ความต่างตรงนั้นสำคัญ

## Section 5: มันทำเงินได้จริงไหม?

คำถามตรงๆ: AI ทำเงินให้ผมไหม?

ใช่

และผมพูดได้โดยไม่ต้องบอกตัวเลข revenue จริง เพราะ ROI ไม่ได้อยู่แค่ที่บรรทัด revenue

มันอยู่ที่สิ่งที่ผมไม่ได้จ่าย:

ผมไม่ต้องจ้างคนเพิ่มเพื่องานที่ agent จัดการได้แล้ว

ผมไม่ต้อง manage ความซับซ้อนที่มาพร้อมการจ้างเหล่านั้น

ผมไม่ต้องรับต้นทุน turnover แบบเดิม

ผมไม่เสียเวลากับ operational drag มากเท่าเดิม

ทีมเล็กลงได้และเคลื่อนที่เร็วขึ้น

Agent รับงาน volume

คนรับงานที่ต้องใช้ judgment และความสัมพันธ์

นั่นคือ ROI จริง

ถ้าคุณเป็น solo founder หรือทีมเล็กในไทยหรือเอเชียตะวันออกเฉียงใต้ leverage ตรงนี้จริง

คุณสามารถทำงานในระดับที่เคยต้องใช้ทีมใหญ่กว่านี้มาก

ไม่จำเป็นต้องเป็น tech company เพื่อใช้สิ่งนี้

แค่ต้องยินดีเรียนรู้วิธีตั้ง setup ให้ถูกต้อง

คนส่วนใหญ่ถามว่า: "ควรใช้ AI tool ตัวไหน?"

คำถามที่ถูกต้องคือ: "ส่วนไหนของธุรกิจที่ซ้ำพอที่จะเปลี่ยนเป็น system ได้?"

ตอบได้: การเลือก tool จะง่ายขึ้นมาก

## Section 6: คำตอบตรงๆ

5 เดือนผ่านมา นี่คือที่ผมยืนอยู่

ต้นทุน hardware จริง แต่เป็นการตัดสินใจครั้งเดียว

ต้นทุน API ต่อเนื่อง แต่ scale ตาม usage ไม่ใช่ headcount

ต้นทุน setup คือส่วนที่คนส่วนใหญ่ข้าม

และนั่นแหละคือเหตุผลที่คนส่วนใหญ่ได้ผลลัพธ์ธรรมดา

มอง AI เหมือนปุ่มวิเศษ: คุณจะผิดหวัง

มองมันเหมือน team member ประเภทใหม่ที่ต้องการ onboarding และ management: คุณสร้างอะไรที่ใช้งานได้จริงได้

ผมจ้างและ manage คนมาเยอะ

AI agent ไม่ใช่คน

มันคือ tool ที่มีรูปแบบความล้มเหลวต่างกันและโครงสร้างต้นทุนที่ง่ายกว่ามาก

สำหรับงานที่ใช่: มันคือการจ้างที่ดีที่สุดครั้งหนึ่งที่ผมทำ

สำหรับงานที่ไม่ใช่: มันแค่เป็นสิ่งรบกวนที่แพง

งานของคุณคือหาว่าอันไหนคืออันไหน สำหรับธุรกิจของคุณ

วิธีเดียวที่จะรู้คือต้องลงมือทดลองจริง

ไม่ใช่อ่านเรื่องมัน

ไม่ใช่ดูวิดีโอ 50 อัน

ไม่ใช่เถียงเรื่อง model บน Twitter

รัน workflow หนึ่งอัน วัดผล แล้วตัดสินใจ
