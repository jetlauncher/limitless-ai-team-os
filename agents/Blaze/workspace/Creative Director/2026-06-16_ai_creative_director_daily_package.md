DEMO/CONTENT (2:30–12:00)
ผมให้ framework ชื่อ 3-Layer AI Resilience.
Layer 1: Prompt portability. อย่าเขียน prompt ที่ผูกกับชื่อฟีเจอร์ของ tool เดียว. ให้แยกเป็น objective, input, constraints, output format. ตัวอย่าง: “คุณคือ sales analyst วิเคราะห์ transcript นี้ สรุป pain points 5 ข้อ, objections 3 ข้อ, next action เป็นตาราง.” Prompt แบบนี้ย้ายจาก Claude ไป ChatGPT, Gemini, หรือ local model ได้ง่าย.
Layer 2: Model fallback map. แบ่งงานเป็น 3 กลุ่ม. งาน creative ใช้ได้หลายโมเดล. งาน reasoning ใช้ top-tier 2 ตัว. งาน sensitive ต้องมี policy ชัดและอาจใช้ private environment. ทำตาราง: primary model, backup model, acceptable quality, cost ceiling. ถ้าค่า subscription สมมติ $20 ต่อคนต่อเดือน ประมาณ 740 บาท อย่ามองว่าแพงอย่างเดียว ให้มองว่าเป็น insurance ของ workflow.
Layer 3: Data export discipline. ทุก output สำคัญต้องเก็บนอกแชท AI: Google Drive, Notion, CRM, repo. ถ้าคุณมี SOP อยู่ใน conversation เดียว แล้ว account มีปัญหา เท่ากับคุณเสีย knowledge base. ตั้งกฎว่า prompt สำคัญ, template, customer insight, campaign result ต้องถูกบันทึกเป็นไฟล์หรือฐานข้อมูล.
Thai SME example: เอเจนซี่โฆษณา 10 คนใช้ AI ทำ proposal. ถ้าใช้ AI ตัวเดียวแล้ว quota หมดวัน pitch ลูกค้า จะเสีย deal. วิธีแก้คือมี template prompt กลาง, model fallback 2 ตัว, และ output archive. ร้านออนไลน์ก็เหมือนกัน: caption, FAQ, response script ต้องอยู่ใน document กลาง ไม่ใช่ในแชทของ admin คนเดียว.
English: Port prompts, map fallback models, and export business-critical data.
