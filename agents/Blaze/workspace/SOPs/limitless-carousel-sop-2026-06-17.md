# Limitless Club Carousel SOP

Version: 2026-06-17
Owner: Blaze / Limitless Club Content System
Reference analyzed: M1 `Bad vs Viral Instagram Carousels` Instagram carousel (`DZqnp8Lms_R`) + Jet/Limitless carousel standards

## 1. Purpose

Create Instagram carousels that are not just beautiful, but easy to read, easy to save, and strategically useful for Thai business owners.

A good Limitless carousel should do three jobs:

1. Stop the scroll with a clear, high-tension idea.
2. Teach one useful business / AI / operating principle.
3. Move the reader toward trust, saves, comments, DMs, or the next Limitless offer.

## 2. Core rule

A carousel is not an article split into slides.

A carousel is a sequence of visual arguments.

Each slide must have one job only.

If a slide has two ideas, split it.
If a slide needs a paragraph, move the explanation into the caption.
If the reader cannot understand the slide in 2 seconds, rewrite it.

## 3. The best carousel format for us

Default structure: 7–10 slides.

1. Cover — big promise, tension, or pattern interrupt.
2. Credibility / context — why this is worth listening to.
3. Problem — what most people get wrong.
4. Reframe — the better way to think about it.
5. Principle / rule 1 — practical, concrete.
6. Principle / rule 2 — practical, concrete.
7. Example / comparison — show the idea in context.
8. Framework / checklist — make it saveable.
9. Takeaway — what to change this week.
10. CTA — save, comment, DM, join, or request the resource.

Shorter version: 6 slides.

1. Hook.
2. Problem.
3. Reframe.
4. Framework.
5. Example.
6. CTA.

## 4. Strategic angle selection

Before writing, choose the carousel type.

### Type A — Mistake correction

Use when the market is doing something wrong.

Example:
- “Most business owners use AI backwards.”
- “The problem is not your content quality. It is your content system.”

Best for: authority, education, comments.

### Type B — Framework / checklist

Use when the reader needs a repeatable process.

Example:
- “The 5-part AI content system every business needs.”
- “Before hiring an AI agency, check these 7 things.”

Best for: saves, DMs, lead magnets.

### Type C — Before / after

Use when showing transformation.

Example:
- “Random posting vs content operating system.”
- “Prompt user vs AI operator.”

Best for: clarity and broad reach.

### Type D — Field note / founder lesson

Use when Jet has a personal business insight.

Example:
- “What I learned after building AI workflows for real teams.”
- “The mistake I see Thai business owners making with automation.”

Best for: premium founder-media feel and trust.

## 5. Writing rules

### Cover slide

The cover must be instantly readable and emotionally specific.

Good cover formulas:

- “หยุดใช้ AI แบบนี้ ถ้าคุณเป็นเจ้าของธุรกิจ”
- “AI ไม่ได้ช่วยธุรกิจที่ยังไม่มีระบบ”
- “5 สิ่งที่ทำให้ Carousel ถูก Save มากขึ้น”
- “Content ที่ดี ไม่ได้เริ่มจากไอเดีย”
- “Prompt ไม่ใช่ Strategy”

Bad cover patterns:

- Generic: “How to use AI for business”
- Too broad: “The future of content marketing”
- Too clever: headline that needs explanation before it lands
- Too dense: more than 2–3 visual lines

### Interior slides

Each interior slide should use this hierarchy:

- Kicker: small label that orients the reader.
- Main headline: one key idea.
- Body: short supporting explanation.
- Visual cue: number, diagram, example, or contrast.

Example:

Kicker: COMMON MISTAKE
Headline: ธุรกิจส่วนใหญ่เริ่มจาก Tool
Body: ทั้งที่ควรเริ่มจาก workflow ที่อยากให้ AI ช่วยทำซ้ำ

### CTA slide

CTA should match the content objective.

- Save CTA: “Save this before building your next carousel.”
- Comment CTA: “Comment ‘SYSTEM’ if you want the checklist.”
- DM CTA: “DM ‘AI OS’ if you want help building this for your business.”
- Community CTA: “Join Limitless Club for business owners building with AI.”

Do not use vague CTAs like “Follow for more” unless the carousel is purely top-of-funnel.

## 6. Readability rules from the M1 reference

The M1 carousel works because it is built for mobile reading first.

### Typography hierarchy

Use large type.

- Cover headline: very large, 2–4 lines.
- Interior headline: large enough to read from a contact sheet.
- Body text: short, not decorative.
- Labels: useful, not clutter.

Reference principle:
- Headline should feel 50pt+ equivalent.
- Body should never become tiny “design filler.”

For our 1080×1350 Thai slides:

- Cover headline: approx 76–96px depending on word count.
- Interior headline: approx 56–76px.
- Body: approx 32–44px.
- Metadata/kicker: approx 20–30px.

If body copy must go below 30px, rewrite or split the slide.

### Safe zones

Keep important text away from the edges.

Minimum safe margins:

- Top: 120–180px depending on design.
- Bottom: 160–220px because Instagram UI and captions can visually crowd the bottom.
- Left: 80–120px.
- Right: 80–120px.

For cover slides that may appear in Reels tab or profile grids, keep the main hook in the center-safe area.

### Layout discipline

Use one dominant object per slide:

- One headline.
- One diagram.
- One comparison.
- One example set.
- One photo.

Do not combine a big headline, long paragraph, 4 screenshots, CTA, and footer on the same slide.

## 7. Visual design standards

Limitless carousels should feel premium, clean, and strategic — not Canva-generic.

### Default Limitless style

Use when creating Jet authority posts.

- Background: black / charcoal / deep graphite.
- Text: warm white / ivory.
- Accent: muted gold / bronze / silver.
- Typography: editorial headline + clean Thai-safe sans.
- Design: high negative space, subtle gridlines, premium restraint.
- Footer: `Limitless Club · @jeditrinupab` or `@jeditrinupab`.

### When to use yellow / high-contrast style

The M1 reference uses yellow and navy because it is educational and punchy. We can adapt the logic, not copy the exact look.

Use high-contrast yellow/gold when:

- The carousel is tactical and meant to be saved.
- The topic is design, hooks, content systems, mistakes, or checklists.
- We want “manual / guide / field note” energy.

Do not overuse bright yellow on every carousel or it will dilute the premium Limitless feel.

### Visual ingredients that work

- Large typography.
- Simple diagrams.
- One screenshot group max per slide.
- Thin boxes around key concepts.
- Arrows used sparingly.
- Strong contrast between headline and body.
- Repeated visual system across slides.

### Visual ingredients to avoid

- Tiny body text.
- Decorative elements that fight the message.
- Too many screenshots.
- Overly complicated gold frames.
- AI-glow visuals with no business meaning.
- Footer too close to the bottom edge.

## 8. Production workflow

### Step 1 — Source intake

Collect one of these:

- Jet voice note / Plaud transcript.
- Signal intel report.
- Client insight.
- Market trend.
- Reference carousel.
- Offer / workshop / lesson.

Then extract:

- One core promise.
- Audience pain.
- Market implication.
- Business example.
- CTA objective.

### Step 2 — Write the text first

Draft the carousel copy before designing.

Required copy package:

- Carousel title.
- Slide-by-slide text.
- Caption draft.
- CTA.
- Visual notes per slide.

Do not render graphics until Jet approves the text when it is a public-facing Jet carousel.

Approval sequence:

Text approval → graphics render → graphics approval → schedule/post.

### Step 3 — Create slide spec

Use a structured slide spec:

```json
{
  "title": "Prompt ไม่ใช่ Strategy",
  "slug": "prompt-is-not-strategy",
  "format": "1080x1350",
  "style": "midnight-luxe-editorial",
  "footer": "@jeditrinupab",
  "slides": [
    {
      "slide": 1,
      "kicker": "AI CONTENT SYSTEM",
      "headline": "Prompt ไม่ใช่ Strategy",
      "body": "ถ้าธุรกิจยังไม่มีระบบ AI จะช่วยให้คุณทำสิ่งเดิมเร็วขึ้นเท่านั้น",
      "visual_note": "large centered headline, dark background, subtle system grid"
    }
  ]
}
```

### Step 4 — Render

Default export:

- 1080×1350.
- JPG or PNG.
- Contact sheet.
- ZIP.
- Local source folder.

For Thai text, prefer HTML/CSS + Chrome rendering or another renderer with proper Thai shaping. Avoid final Thai text rendering in Pillow unless it has been tested because diacritics and line spacing can break.

### Step 5 — QA

QA every carousel before delivery.

Required checks:

- Hook is clear in 2 seconds.
- Every slide has one idea.
- Thai reads naturally.
- No text is cut off.
- Text is readable at phone size.
- Footer / handle is safe and not too low.
- Contact sheet shows all slides.
- Final files are exactly 1080×1350.
- ZIP opens correctly.

Extra checks for dense slides:

- Inspect the individual slide, not only the contact sheet.
- If a slide looks crowded, shorten the copy instead of shrinking the font.

### Step 6 — Archive

For every serious carousel package, save:

- `slides.json`
- rendered images
- contact sheet
- ZIP
- caption
- source/reference notes
- QA notes

Archive in Blaze assets or Notion Work Output if it is a durable SOP, strategy, or content package.

## 9. Carousel scorecard

Score each draft from 1–5.

| Category | Question | Pass standard |
|---|---|---|
| Hook | Would the target audience stop? | Specific tension, not generic topic |
| Utility | Is there a usable takeaway? | Reader can apply or save it |
| Sequence | Does each slide advance the argument? | No repeated filler slides |
| Readability | Can it be read on mobile fast? | Large type, short lines |
| Brand | Does it feel like Limitless / Jet? | Premium, practical, not hypey |
| CTA | Is the next action clear? | Save, comment, DM, join, or book |
| QA | Is it production safe? | No clipping, correct dimensions |

A carousel should not ship below 4/5 on Hook, Utility, and Readability.

## 11. Caption formula

A good carousel caption should deepen the slide, not repeat every slide.

Structure:

1. Restate the core insight.
2. Add one business-owner example.
3. Give 3–5 practical bullets.
4. CTA.

Example:

```text
Most businesses do not need more content ideas.
They need a content operating system.

Because random ideas create random output.
A system creates consistency.

Start with:
- 3 recurring audience problems
- 2 repeatable content formats
- 1 weekly batch day
- 1 QA checklist before posting
- 1 metric to review every week

Save this before asking AI for another list of ideas.
```

## 12. Final operating principle

The best Limitless carousel should feel like:

- A premium business note.
- A simple operating lesson.
- A founder-level reframe.
- A post worth saving.

Not a decoration.
Not a quote card.
Not an article squeezed into 10 images.

If the idea is not strong, better design will not save it.
If the idea is strong but unreadable, the market will not receive it.

Win with: strong idea → clean sequence → large readable design → tight CTA → QA before publishing.
