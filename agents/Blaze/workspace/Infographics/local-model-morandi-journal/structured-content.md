# คู่มือ Local Model สำหรับเจ้าของธุรกิจ

## Overview
อินโฟกราฟิกอธิบายว่า Local Model คืออะไร เหมาะกับงานแบบไหน และเจ้าของธุรกิจควรตัดสินใจใช้ร่วมกับ Cloud AI อย่างไร

## Learning Objectives
The viewer will understand:
1. Local Model คือ AI ที่รันบนคอมเรา ไม่ใช่บน Cloud
2. เหตุผลทางธุรกิจ: ลดต้นทุนซ้ำ คุมข้อมูล ทำงานออฟไลน์ได้
3. ข้อแลกเปลี่ยน: ต้องมีเครื่องแรง RAM เยอะ และโมเดลอาจไม่เก่งที่สุด
4. วิธีคิดแบบผู้บริหาร: Cloud เพื่อความเร็ว / Local เพื่อการควบคุม

**Key Concept**: LM Studio ทำให้ Local Model ใช้ง่ายขึ้น คล้ายมี ChatGPT บนเครื่อง

## Section 4: กฎเรื่องความแรง

**Key Concept**: โมเดลยิ่งใหญ่ยิ่งฉลาด แต่กิน RAM และพื้นที่มากขึ้น

**Content**:
- ตัวอย่างโมเดลในคลิป: Qwen 3.6 35B parameters
- Parameters ยิ่งเยอะ ยิ่งฉลาด
- ใช้ RAM และพื้นที่มากขึ้น
- ถ้าจะซื้อคอมใหม่สำหรับ Local Model ให้เน้น RAM เยอะที่สุดก่อน

**Visual Element**:
- Type: hand-drawn scale
- Subject: parameters scale, RAM bar, hard disk icon
- Treatment: terracotta number callout

**Text Labels**:
- Headline: "กฎง่ายๆ"
- Labels: "Parameters มาก = ฉลาดขึ้น", "RAM มาก = รันได้ดีขึ้น", "พื้นที่ต้องพอ"

## Section 5: Cloud vs Local

**Key Concept**: ไม่ใช่เลือกอย่างใดอย่างหนึ่ง แต่ใช้ให้ถูกงาน

**Content**:
- Cloud AI ยังเหมาะกับงานที่ต้องการรุ่นเก่งที่สุดและความเร็ว
- Local Model เหมาะกับงานที่ต้องการควบคุมข้อมูล ใช้ซ้ำเยอะ และลดต้นทุน

**Visual Element**:
- Type: split decision card
- Subject: cloud icon vs laptop icon
- Treatment: rounded Morandi cards with check marks

**Text Labels**:
- Headline: "ใช้คู่กันให้ถูกงาน"
- Cloud: "Cloud = เร็ว + เก่งสุด"
- Local: "Local = คุมข้อมูล + ลดต้นทุน"

---

## Section 6: Founder Decision Rule

**Key Concept**: ตัดสินใจด้วยความเสี่ยงข้อมูลและความถี่ของงาน

**Content**:
- งานข้อมูลสำคัญ / ทำซ้ำบ่อย → Local
- งานต้องการคุณภาพสูงสุด / เร่งด่วน → Cloud
- เป้าหมายคือสร้าง AI Operating System ที่คุมได้เอง

**Visual Element**:
- Type: sticky-note decision flow
- Subject: two arrows leading to Local or Cloud
- Treatment: washi tape, dotted arrows, founder notebook

**Text Labels**:
- Headline: "สูตรจำง่าย"
- Labels: "ข้อมูลสำคัญ → Local", "งานด่วน/ยาก → Cloud", "AI OS ที่คุมได้เอง"
