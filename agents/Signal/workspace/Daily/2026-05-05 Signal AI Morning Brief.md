### 3) Agent operations and security are becoming a real stack
- **What changed:** AWS announced AgentCore Optimization preview: production traces -> recommendations -> batch evaluation -> A/B testing. Cisco announced intent to acquire Astrix Security to secure non-human identities and AI agents across the enterprise.
- **Why it matters:** Production agents need the same discipline as software systems: observability, evals, rollout/rollback, identity, authorization, and incident response.
- **Who should care:** Anyone deploying agents into customer operations, internal tooling, or regulated workflows.
- **Action/angle:** Teach/build the “agent ops checklist”: trace every task, eval before release, A/B improvements, approve sensitive actions, secure every agent identity.
- **Sources:** https://aws.amazon.com/blogs/machine-learning/introducing-the-agent-quality-loop-agentcore-optimization-now-in-preview/ ; https://blogs.cisco.com/news/cisco-announces-intent-to-acquire-astrix-security

## Founder/operator implications
1. **Enterprise AI is shifting from demos to implementation capacity.** Services, workflow mapping, and change management are becoming defensible go-to-market layers.
2. **Distribution is moving into existing enterprise platforms.** Builders should design for governance surfaces customers already trust: AWS, Databricks, identity stacks, SIEM, and approval workflows.
3. **Agent reliability/security is now a buyer concern.** “Can your agent be observed, evaluated, authorized, and rolled back?” will matter more than raw model choice.
