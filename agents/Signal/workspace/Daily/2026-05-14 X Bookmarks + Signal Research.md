### 1. Agent security is becoming infrastructure, not an afterthought
- What changed: AWS published a May 13 post with Cisco AI Defense on securing enterprise MCP and A2A deployments. It frames the problem as visibility across deployed tools/agents, security review velocity, and missing audit trails for compliance. Google separately framed Android as moving from OS to intelligence system, with Gemini app automation controlled by opt-ins, app-specific permissions, purchase confirmation, progress indicators, and Privacy Dashboard history.
- Why it matters: This directly upgrades Jet’s bookmark cluster around Hermes/OpenClaw/Manus/cloud computers: the winning agent stack is not just “can it automate?” but “can it automate with permissions, logs, auditability, and user-visible control?”
- Who should care: Limitless Club operators, AI audit/service founders, educators teaching safe agent workflows, SMB/enterprise AI leads.
- Recommended action/angle: Turn the next Limitless exercise into an Agent Control Checklist: allowed apps/tools, MCP/A2A inventory, human approval points, payment limits, run logs, rollback path, and privacy/dashboard proof.

### 2. Agent commerce and authenticated tool access keep hardening into cloud primitives
- What changed: AWS’s official Bedrock AgentCore Payments post says agents are moving beyond assistants into API/MCP/agent coordination and may discover, evaluate, and pay for resources inside a single execution loop. AWS also made its managed AWS MCP Server generally available so coding agents can use authenticated AWS services through a fixed tool set rather than broad credentials.
- Why it matters: This is the clearest confirmation of the “agent operating layer” thesis from Jet’s bookmarks: identity, payments, authenticated tools, and policy boundaries are becoming the defensible layer around models.
- Who should care: Founders building automations, agencies selling AI workflow retainers, dev teams putting Claude/Codex/Cursor into production, operators worried about spend and credentials.
- Recommended action/angle: For client offers, stop selling “chatbot automation.” Sell governed execution: tool access map, spend/payment policy, scoped credentials, logs/evals, and approval gates.
- Sources:
