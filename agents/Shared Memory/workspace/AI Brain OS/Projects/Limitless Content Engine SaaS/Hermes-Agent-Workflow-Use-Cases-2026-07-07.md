## Executive summary

The current Hermes agent system is no longer just a chat assistant. It is a multi-agent operating layer for Jet / Limitless that handles:

- Executive briefing and chief-of-staff routines
- Revenue, payments, Airtable, Stripe, and Meta Ads monitoring
- Content research, generation, repurposing, and approval workflows
- AI news and X intelligence monitoring
- Course and class operations
- Agent memory, Obsidian/Notion synchronization, and self-improving knowledge loops
- Website, Vercel, dashboard, and automation QA
- Voice, LINE OA, Telegram, and phone-agent interfaces
- Premium media and creative asset production

The sales framing should not be low-ticket SaaS. The marketable offer is: **AI Team OS by Limitless - premium agent workflows installed inside a business, with human approval gates and founder-level taste.**

## Notes and current caveats

- Some workflows are active and running on cron today.
- Some workflows are built but paused because they were superseded or noisy.
- A few integrations currently need maintenance, especially X-to-Obsidian and some OAuth-dependent YouTube/GSC flows.
- The sales page should present workflows as proven capabilities, not all as fully self-serve SaaS features.
- For client delivery, start with premium internal operation and install workflows manually before exposing a customer dashboard.
