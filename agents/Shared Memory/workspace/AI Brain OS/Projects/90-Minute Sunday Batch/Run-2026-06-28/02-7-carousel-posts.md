## 2. AI Agent ไม่ใช่ Chatbot — มันคือพนักงานที่ต้อง Onboard

**Source:** Plaud 2026-06-26 Hermes Agent setup

**Pillar:** AI Team OS / agents as employees

**Caption:** If you would not hire a human without JD, tools, examples, KPI, and review rhythm, do not expect an AI agent to perform with one vague prompt. Treat agents like employees: role, data access, queue, log, memory, and human control.

**Slides / value points:**
1. Hook: Prompt ไม่ใช่พนักงาน AI — prompt เป็นแค่ใบสั่งงาน
2. Chatbot answers. Agent does work.
3. Every agent needs one clear role first
4. JD: what this agent owns and what it must never touch
5. Tools: what systems it can read/write
6. Memory: what it should remember after each run
7. Queue: where tasks arrive and status is visible
8. Guardrails: when it must ask the human
9. Review: score output before automation
10. CTA: onboard one AI employee this week

**Caption:** If you classify AI as software, every subscription feels expensive. If you classify it as payroll leverage, the question changes: which recurring task, report, response, draft, or analysis does it buy back?

5. Example: 600 comments become an approved response workflow
6. Example: 1 workshop becomes 30 content assets
7. Founder math: compare against your hourly value
8. Do not replace people. Replace waste.
9. Budget AI by task, not by tool name
10. CTA: list 5 payroll tasks AI should buy back

## 5. Don’t Automate Chaos — Promote the Workflow First

**Source:** Plaud Insight Bank + 2026-06-26/27 agent setup

**Pillar:** AI Team OS / agents as employees

**Caption:** Automation is not the first step. First teach the workflow. Then test. Then correct. Then turn it into SOP/Skill. Then schedule. Otherwise you scale mistakes while feeling productive.

**Slides / value points:**
1. Hook: Don’t automate chaos. Train the AI before giving it the office keys.
2. Most owners skip straight to automation
3. But messy workflow + automation = faster mess
4. Stage 1: Run manually with AI assistance
5. Stage 2: Review the output like a manager
6. Stage 3: Write the correction into SOP/Skill
7. Stage 4: Run again and compare
8. Stage 5: Only then schedule it
9. Rule: trusted workflow before cron job
10. CTA: promote one workflow this week
