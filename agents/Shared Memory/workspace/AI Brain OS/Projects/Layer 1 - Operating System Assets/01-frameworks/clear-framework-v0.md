# CLEAR Framework — Prompt / Request Builder v0

> Workspace note: I found references to CLEAR as a founder framework, but not a final canonical acronym/spec. This is a practical v0 for member-portal use and should be reviewed before becoming official doctrine.

## CLEAR = Context, Limits, Examples, Action, Review
CLEAR turns a rough request into a usable work order.

## C — Context
Give the business situation: audience, goal, prior context, source material, and business result.

## L — Limits
Set constraints: approvals, tone, length, sensitive data, assumptions not allowed.

## E — Examples
Show what good looks like: past winning post, good email, best proposal, brand voice sample, preferred report, bad example to avoid.

## A — Action
Ask for a specific output: draft, summarize, compare, score, rewrite, analyze, checklist, or plan.

## R — Review
Tell AI how the human will judge the result: accuracy, usefulness, brand fit, risk level, next-action clarity, decision-readiness.

## CLEAR template
```text
Context:
[Business situation, audience, source material, goal]

Limits:
[Constraints, approvals, sensitive data, tone, length]

Examples:
[What good looks like / what to avoid]

Action:
[The exact output to produce]

Review:
[How to score the result and what to flag]
```

## One-line version
"Given [context], within [limits], using [examples], produce [action/output], and self-review against [criteria]."
