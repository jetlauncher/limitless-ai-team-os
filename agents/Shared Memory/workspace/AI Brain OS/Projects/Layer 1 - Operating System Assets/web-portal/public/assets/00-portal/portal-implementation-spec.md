# Portal Implementation Spec — Layer 1 Operating System

## Goal
Turn Layer 1 into a member portal that feels like an operating dashboard, not a folder of PDFs.

## Core navigation
1. Start Here
2. Strategy Map
3. Build Your First AI Employee
4. Prompt Library
5. Templates
6. Weekly Operating Rhythm
7. Completion Checklist

## Lesson page structure
Each lesson should have:
1. Concept in one sentence
2. Why it matters
3. Template / worksheet
4. Example
5. 15-minute action
6. Upload / save checkpoint

## Completion logic
A member completes Layer 1 only when all are true:
- Golden Triangle worksheet filled
- KOI role brief filled
- One CLEAR prompt used
- Approval Boundary Map filled
- One draft output produced
- Final human review completed
- One correction saved back into Knowledge Loop

## Design direction
Premium business OS: dark navy, warm gold, clean cards, operator dashboard feel. Avoid playful SaaS toy visuals. This should feel like a serious internal business system.
