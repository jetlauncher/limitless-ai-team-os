# Approval Boundary Map

## Always require approval
- [ ] Sending external emails/messages
- [ ] Social publishing
- [ ] Payments/refunds/invoices
- [ ] Deleting data
- [ ] Changing automations
- [ ] Deployments
- [ ] Accessing large private datasets
- [ ] Legal/financial/medical advice
- [ ] Pricing exceptions or discounts

## Safe autonomous actions
- [ ] Read approved docs
- [ ] Summarize
- [ ] Draft
- [ ] Categorize
- [ ] Create internal notes
- [ ] Prepare reports
- [ ] Update pre-approved internal trackers

## Escalation triggers
Angry customer, high-value lead, refund/payment issue, legal/financial/medical question, missing information, data conflict, confidence below threshold, or new scenario not in Knowledge Loop.
