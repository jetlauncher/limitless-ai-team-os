# Start Here — The AI Operating System

## The shift
Most companies do not need more AI tools. They need an operating system.

A tool answers a prompt. An operating system turns company knowledge into repeatable output.

## Simple definition
An AI Operating System combines:
1. **Knowledge** — what the company knows, believes, sells, promises, and has learned from real work
2. **Instructions** — role briefs, rules, workflows, approval gates, quality standards
3. **Outputs** — reports, drafts, follow-ups, content, dashboards, proposals, checklists
4. **Review loop** — human judgment that corrects the system and improves the next run

## Member outcome
By the end of Layer 1, a member should have:
- One company AI strategy map
- One Knowledge Loop checklist
- One first AI employee role brief
- One working 10-80-10 delegation template
- One department prompt pack they can use immediately
- One approval boundary map
- One 7-day implementation plan

## Operating principle
Do not start by asking, "Which AI tool should we use?"
Start by asking:
1. What work repeats every week?
2. What knowledge does this work require?
3. What output should AI produce?
4. What does a human still need to approve?
5. How will the result improve the system next time?
