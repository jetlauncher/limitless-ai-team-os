# Content Production Protocol

## Trigger

Input can be:

- Type A — Spark: one-liner, short voice memo, quick note.
- Type B — Brain dump: long text, Google Doc, long voice memo.
- Type C — Recording: podcast, workshop, long video/audio/transcript.

## Parallel workflow

```text
Jedi input
  ├─ Agent 1: Long-form Director
  ├─ Agent 2: Short-form Agent
  └─ Agent 3: Clipping Agent / Miner
```

## Handoff rules

1. Long-form Director broadcasts the chosen long-form **title/angle** to the other agents.
2. Clipping Agent sends clip map before Short-form Agent finalizes clip-derived shorts.
3. Short-form Agent can immediately create idea-native shorts from the seed while waiting.
4. All outputs go into the same idea bundle.

## Weekly rhythm

| Day | Activity |
|---|---|
| Mon | Jedi drops 1–3 idea seeds. Agents generate planning outputs. Jedi picks packaging in ≤5 min. |
| Tue–Wed | Recording: podcast/training/solo doc video. Prep should be ≤10 min using Agent 1 brief. |
| Wed–Thu | Agent 3 mines recording. Agent 2 builds platform-native batch. Agent 1 finalizes publish package. |
| Fri | Long-form publishes. Short-form rollout begins: 1–2 posts/platform/day. |
| Sun | Learning loop: update proven-content-log with performance findings. |

## Success metrics

Track:

1. Long-form CTR.
2. Average view duration / retention.
3. Short-form views, saves, shares, skip rate.
4. Comment-keyword leads.
5. Cohort/workshop enrollments attributed.
6. Jedi hands-on time per published minute.

Governing metric: **audience quality × views → enrollments**.
