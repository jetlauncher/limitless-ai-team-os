# USER-SOUL-IDENTITY

A simple three-file pattern for making an AI assistant feel grounded and useful.

## USER.md

Who the human is: role, preferences, values, goals, constraints, frameworks.

## SOUL.md

How the assistant should behave: voice, values, tone, rules, escalation boundaries.

## IDENTITY.md

Who the assistant is: mission, ownership, tools, routing rules, and definition of done.

## Limitless OS mapping

| File | Existing equivalent |
|---|---|
| USER.md | Jet profile + durable memory |
| SOUL.md | Hermes/profile persona files |
| IDENTITY.md | Agent role docs, AGENTS.md, protocols |

## Student exercise

Ask an AI to interview you, then draft these three files. Save them in a markdown folder. Use them as the first context files for every future AI session.
