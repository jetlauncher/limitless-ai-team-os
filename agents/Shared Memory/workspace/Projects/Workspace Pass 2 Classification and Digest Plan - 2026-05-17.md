## Proposed Jet Brain Digest

### Goal
Make the full system usable daily by surfacing only what changed, what matters, where the source lives, and what needs Jet’s decision. No giant logs. No repeated summaries of old files.

### Daily Telegram digest — 08:00, optional 17:30 mini-digest
1. **Today’s command center:** 3–5 active links/tasks/dashboards.
2. **Revenue/Ops:** Airtable revenue changes, payment/register anomalies, urgent email/calendar items.
3. **Agent fleet:** cron failures, stuck jobs, noisy agents, missing memory notes.
4. **Content/Research:** top 3 usable ideas/assets from Blaze/Signal/Kaijeaw/Zegna.
5. **Teaching/Product:** new or changed course/workshop/app assets.
6. **Needs Jet decision:** max 3 bullets, each with a recommended default.
7. **Sources touched:** short file/link citations.

### Weekly digest — Sunday evening attachment
Wins, open loops, dashboard/app health, revenue trend, content produced, teaching assets, project drift, cleanup recommendations, next week priorities. Deliver Markdown/PDF.

### Rules
- Digest from changes since last digest, not the whole vault.
- Always cite source paths/URLs.
- Max 10 bullets in Telegram unless urgent.
- Use Airtable for revenue truth; local notes are supporting context.
- Keep GBrain read-only until it beats Jet Brain v0 retrieval benchmark.
- Never include secrets, raw logs, auth files, hidden config, or token values.

## Automation recommendation

1. Nightly local scanner (`deliver: local`, no-agent script) updates classification CSV/JSON and changed-file state.
2. Morning Kelly digest (`deliver: origin`) reads scanner output + Airtable + cron health and sends Telegram digest.
3. Weekly deep digest (`deliver: origin`) creates a Markdown/PDF attachment with project/app/agent drift.
4. Monthly benchmark runs `jet-brain-retrieval-benchmark.md` before any GBrain import expansion.
