# AI Team OS Command Room Checklist v0

A student-ready and operator-ready checklist for running an AI agent team without losing control.

## The 6-step loop

### 1. Capture
- **Action:** Put every request, idea, and artifact into one visible inbox before asking agents to act.
- **Where it lives:** `Shared Memory/Daily + agent Scratchpad inboxes`
- **Done when:** there is visible evidence in the workspace.

### 2. Route
- **Action:** Assign work by agent role: Kelly decides, Signal researches, Blaze packages, Bolt builds, Qwen preprocesses.
- **Where it lives:** `Agent registry and handoff note`
- **Done when:** there is visible evidence in the workspace.

### 3. Constrain
- **Action:** State explicit no-go actions: no deploys, purchases, deletes, emails, or posts without approval.
- **Where it lives:** `Safety checklist block`
- **Done when:** there is visible evidence in the workspace.

### 4. Build
- **Action:** Require a tangible local artifact: HTML, markdown pack, script, dashboard, prototype, or worksheet.
- **Where it lives:** `Bolt/Builds/YYYY-MM-DD/<slug>/`
- **Done when:** there is visible evidence in the workspace.

### 5. Verify
- **Action:** Open the artifact, syntax-check scripts, count key files, and read back acceptance criteria.
- **Where it lives:** `Verification log`
- **Done when:** there is visible evidence in the workspace.

### 6. Handoff
- **Action:** Write local daily notes plus shared daily bullets so the next agent sees what changed.
- **Where it lives:** `Shared Memory/Daily/YYYY-MM-DD.md`
- **Done when:** there is visible evidence in the workspace.

## Classroom exercise
1. Pick one business workflow.
2. Write one request for Kelly, one request for Signal, and one request for Bolt.
3. Add a safety constraint.
4. Define the artifact that proves the run is done.
5. Write the shared handoff bullet the next agent should read.

## Operator acceptance criteria
- A non-technical student can understand the loop in under 3 minutes.
- Jet can reuse it as a slide, handout, or internal checklist.
- The HTML file opens locally and prints cleanly.
